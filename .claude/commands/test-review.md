Проверь качество тестов.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/query`

## Шаги

1. Прочитай `tasks/$ARGUMENTS/acceptance.md` и тесты фичи
2. Проверь по чеклисту:

**Покрытие:**
- каждый сценарий из `acceptance.md` покрыт тестом
- нет тестов для сценариев которых нет в acceptance

**Качество:**
- тест проверяет поведение, а не реализацию
- нет моков там где можно использовать реальную зависимость
- моки оправданы — только для внешних сервисов или медленных операций
- нет тестов проверяющих очевидное (`assert(1 == 1)`)
- тест падает по правильной причине если убрать реализацию

**Изоляция:**
- тесты не зависят от порядка запуска
- тесты не оставляют состояния после себя (cleanup после каждого теста)

3. Классифицируй найденные проблемы и запиши в `tasks/$ARGUMENTS/status.md`:
   - если чисто — `test-review: done`
   - если есть проблемы — `test-review: failed` + список проблем
4. Выведи итог пользователю

## Важно

Не исправлять тесты самостоятельно — только описывать проблемы.
Решение о том куда двигаться дальше принимает оркестратор.
