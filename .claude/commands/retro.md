_Режим: **Харли Куин**. Выполняется оператором напрямую._

Ретроспектива фичи: recall → обсуждение командой → список проблем и решений в RETRO.md.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/connection`

## Требования к окружению

Нужен `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` (agent teams — experimental). Если не задан — добавь в `.claude/settings.json`:
```json
{ "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

## Шаг 1: Recall

Вызови `/recall-notes-and-complaints $ARGUMENTS`.

Дождись завершения — все `notes-*.md` и `complaints-*.md` должны быть обновлены.

## Шаг 2: Участники

Прочитай `tasks/$ARGUMENTS/` — определи кто участвовал:
- Есть `spec.md`, `acceptance.md`, `*-fix-*.md` от спеки → **Танк** (`Танк`), файл: `notes-танк.md`
- Есть код фичи, `code-fix-*.md` → **Ада** (`Ада Лавлейс`), файл: `notes-ада.md`
- Есть `*-review-*.md` → **Гримм** (`Гримм`), файл: `notes-гримм.md`
- Есть тест-файлы, `test-fix-*.md`, `test-run-report.md` → **Кроули** (`Кроули`) и **Азирафаль** (`Азирафаль`), файлы: `notes-кроули.md`, `notes-азирафаль.md`

Прочитай `tasks/$ARGUMENTS/notes-user.md` и `tasks/$ARGUMENTS/complaints-user.md` если существуют — они войдут в общий контекст ретро.

## Шаг 3: Создание команды

Создай team через TeamCreate:
- `team_name`: `retro-<последняя часть пути>` (для `store/connection` → `retro-connection`)
- `description`: `Ретроспектива фичи $ARGUMENTS`

Создай через TaskCreate одну задачу для каждого участника:
- Название: `<агент>: подготовить выжимку наблюдений по фиче $ARGUMENTS`
- Назначь конкретному тиммейту после спауна

## Шаг 4: Спаун тиммейтов

Для каждого участника (из шага 2) спауни тиммейта через Agent tool:
- `team_name`: имя созданной команды
- `name`: имя агента строчными буквами (`танк`, `ада`, `гримм`, `кроули`, `азирафаль`)
- `subagent_type`: тип агента (`Танк`, `Ада Лавлейс`, `Гримм`, `Кроули`, `Азирафаль`)
- `mode`: `bypassPermissions`
- `prompt`: персонализированный промпт (см. ниже)

**Промпт каждому тиммейту** (подставь конкретное имя файла):

> Ты участвуешь в ретроспективе фичи `$ARGUMENTS` как тиммейт команды ретро.
>
> Прочитай свои артефакты:
> - `tasks/$ARGUMENTS/notes-<имя>.md`
> - `tasks/$ARGUMENTS/complaints-<имя>.md`
>
> Если существуют — также прочитай `tasks/$ARGUMENTS/notes-user.md` и `tasks/$ARGUMENTS/complaints-user.md`.
>
> Подготовь краткую выжимку (3-5 пунктов): что болело сильнее всего, какие паттерны видишь, что предлагаешь изменить. Отправь выжимку лидеру.
>
> Затем жди вопросов от Харли. Отвечай развёрнуто, уточняй детали. Можешь писать другим тиммейтам напрямую если нужно уточнить смежную проблему.

## Шаг 5: Сбор и обсуждение

Дождись выжимок от всех тиммейтов (приходят автоматически).

### 5а. Запрос конкретики

Отправь каждому тиммейту через SendMessage:

> «Назови 2-3 главных проблемы этой фичи — конкретно, с примером из того что произошло.»

### 5б. Группировка

Объедини полученные проблемы по категориям:
- **Процессные** — оркестрация, переходы между шагами, ретраи, флоу
- **Инструментальные** — что мешало инструментарий, ограничения платформы
- **Требования** — нечёткие требования, пропущенные случаи, угадывание в спеке
- **Командные** — взаимодействие между агентами, потеря контекста, передача данных

### 5в. Уточнение

По каждой значимой проблеме — отправь уточняющий вопрос автору:

> «[Проблема] — что именно произошло? Как это проявилось в твоей работе? Что бы помогло?»

### 5г. Стратегические вопросы → пользователь

Если возникли вопросы требующие решения пользователя (приоритеты, ограничения, архитектурные выборы) — задай пользователю напрямую, жди ответа прежде чем двигаться дальше.

### 5д. Выработка решений

Для каждой группы проблем — предложи решение. Проверь с тиммейтами через SendMessage:

> «Для [категории проблем] предлагаю: [решение]. Видите подводные камни? Что упустила?»

Скорректируй по обратной связи. Зафиксируй финальные формулировки.

## Шаг 6: RETRO.md

После завершения обсуждения запиши `tasks/$ARGUMENTS/RETRO.md`:

```markdown
# Ретро: $ARGUMENTS

_Дата: <дата>_  
_Участники: <список агентов>_

## Что шло хорошо

- ...

## Проблемы

### <Категория>

- **<Проблема>** — источник: <агент>
  <Детали из обсуждения>

## Предлагаемые решения

| Проблема | Решение | Приоритет |
|---|---|---|
| ... | ... | высокий / средний / низкий |

## Берём в работу

- [ ] <задача> (решает: <проблему>)
- [ ] ...

## Открытые вопросы

- ...
```

## Шаг 7: Завершение команды

1. Отправь каждому тиммейту shutdown через SendMessage: `message: {type: "shutdown_request"}`
2. Дождись завершения всех тиммейтов
3. Вызови TeamDelete

Покажи пользователю итоговую сводку: сколько проблем найдено по категориям, сколько решений предложено, что берём в работу — это actionable backlog.
