Проверь что тестовые сценарии полностью покрывают спеку.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/query`

## Шаги

1. Прочитай `tasks/$ARGUMENTS/spec.md` и `tasks/$ARGUMENTS/acceptance.md`
2. Если `acceptance.md` не существует — сообщи что сначала нужен `/acceptance-write` и остановись
3. Проверь по чеклисту:
   - каждый happy path из ФТ имеет хотя бы один сценарий
   - граничные условия покрыты
   - corner-cases из spec-review покрыты
   - НФТ которые можно проверить тестом — покрыты
   - нет сценариев проверяющих очевидное (1=1, "возвращает то что передали")
   - сценарии конкретны — нет расплывчатых "Then система работает корректно"

4. Классифицируй найденные проблемы:
   - **Пробел в acceptance** — поведение описано в спеке, но сценарий не написан
   - **Неопределённость в спеке** — сценарий невозможно написать, поведение не описано
   - **Противоречие в спеке** — два требования несовместимы

5. Запиши результат в `tasks/$ARGUMENTS/status.md`:
   - если проблем нет — `acceptance-review: done`
   - если есть проблемы — `acceptance-review: failed` + список проблем с классификацией

6. Выведи итог пользователю:
   - если чисто — "Acceptance готов к написанию тестов"
   - если есть проблемы — показать список с классификацией каждой

## Важно

Не исправлять acceptance.md и spec.md самостоятельно — только описывать проблемы.
Решение о том куда двигаться дальше принимает оркестратор.
