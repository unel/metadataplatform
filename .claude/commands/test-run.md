Запусти тесты и проанализируй падения.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/query`

## Шаги

1. Определи команду запуска по стеку фичи:
   - Go: `go test ./...` в папке фичи, или `go test -run TestName` для конкретного теста
   - Frontend юниты: `bun run test`
   - Frontend e2e: `bun run test:e2e`
   - Storybook компоненты: `bun run test:storybook`
2. Запусти тесты
3. Если все прошли:
   - обнови `tasks/$ARGUMENTS/status.md` — `test-run: done`
   - сообщи пользователю результат (сколько тестов прошло)
4. Если есть падения — для каждого упавшего теста:
   - прочитай текст ошибки
   - определи причину:
     - **баг в коде** — реализация не соответствует ожидаемому поведению
     - **баг в тесте** — тест написан некорректно
     - **проблема окружения** — нет соединения с БД, не запущен сервис и т.п.
   - опиши конкретно что не так и где
5. Запиши в `tasks/$ARGUMENTS/status.md` — `test-run: failed` + список падений с причинами
6. Выведи итог пользователю

## Важно

Не исправлять код или тесты самостоятельно — только диагностировать.
Решение о том куда двигаться принимает оркестратор.
