Собери нефункциональные требования для фичи и добавь в spec.md.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/query`

## Шаги

1. Прочитай `CLAUDE.md` и `tasks/$ARGUMENTS/spec.md` (ФТ должны быть уже написаны)
2. Если `spec.md` не существует или не содержит ФТ — сообщи что сначала нужен `/spec-ft`
3. Составь нефункциональные требования по категориям ниже — только те что релевантны данной фиче, не добавляй ради галочки
4. Покажи черновик пользователю и жди подтверждения или правок
5. После подтверждения — запиши в `tasks/$ARGUMENTS/spec.md` секцию `## Нефункциональные требования`
6. Обнови `tasks/$ARGUMENTS/status.md` — этап `spec-nft: done`
7. **Каскадный сброс:** если в `status.md` уже было `spec-review: done` — выставь `spec-review: pending`, а все последующие шаги (`spec`, `acceptance-write`, `acceptance-review`, `test-write`, `test-review`, `code-write`, `code-review`, `build`, `test-run`) переведи в `needs-recheck`

## Категории НФТ (бери только релевантные)

- **Производительность** — время ответа при типичной/пиковой нагрузке, ожидаемый объём данных
- **Надёжность** — поведение при недоступности зависимостей, атомарность, частичный сбой
- **Параллелизм** — конкурентные запросы, блокировки, идемпотентность
- **Безопасность** — валидация входных данных, ограничения доступа
- **Наблюдаемость** — что логируется, метрики
- **Сопровождаемость** — насколько легко изменить/расширить, явные точки расширения, что не должно быть жёстко зашито
- **Читаемость** — код понятен без дополнительного контекста, публичные интерфейсы задокументированы, документация синхронизирована с кодом и не устаревает
- **Ограничения реализации** — зависимости, ограничения платформы

## Формат секции в spec.md

```markdown
## Нефункциональные требования

### Производительность
- ...

### Надёжность
- ...
```

## Важно

Если возникает архитектурный вопрос — не придумывай сам, опиши контекст, варианты и жди ответа:

```
## Нужно решение

**Контекст:** ...

**Варианты:**
1. ... **Плюсы:** ... **Минусы:** ...
2. ... **Плюсы:** ... **Минусы:** ...

**Выбери вариант (1/2/...) или опиши своё решение:**
```
