## Шаги

### 1. Найди все ветки с задачами
Через `/git` получи список веток. Отфильтруй: `feature/*`, `bug/*`, `idea/*`, плюс текущая ветка если не входит в список.

### 2. Собери сводки параллельно
Для каждой найденной ветки запусти `/summarize-branch <ветка>` — параллельно.

Также прочитай `tasks/*/status.md` из текущей ветки напрямую — там могут быть задачи без отдельной ветки.

### 3. Агрегируй
Объедини все сводки, классифицируй задачи:
- **В процессе** — есть незавершённые шаги или зафейленные статусы
- **Застряли** — `implement-iterations ≥ 3` или `needs-rfc`
- **Готовы к следующему шагу** — все предыдущие шаги done, следующий не начат
- **Только идеи** — spec не начата

### 4. Предложение
Покажи пользователю сводку. Для каждой задачи "в процессе" или "готова к следующему шагу" — показывай полную цепочку оставшихся шагов (из сводки `summarize-branch`), не только первый шаг.

Приоритет отображения: "в процессе" → "готовы к следующему шагу" → "застряли" → "только идеи".
Застрявшие — отдельно с пояснением почему.

Покажи сводку и **сразу вызови** `/return-to-feature <задача наивысшего приоритета>` — без ожидания подтверждения.
