Восстанови контекст и продолжи работу над конкретной задачей.

Аргументы: `$ARGUMENTS` — путь к задаче, например `feature/store-query`

## Шаги

### 1. Восстановление контекста
Прочитай:
- `tasks/$ARGUMENTS/status.md` — что сделано, что упало
- `tasks/$ARGUMENTS/spec.md` — напомни себе цель
- `tasks/$ARGUMENTS/deps.md` — зависимости
- Последние коммиты в текущей ветке через `/git`

### 2. Ветка
Через `/git` проверь существует ли ветка для этой задачи (`feature/$ARGUMENTS`, `bug/$ARGUMENTS` и т.д.):
- Если ветка есть — переключись на неё
- Если ветки нет и мы на `main`/`master` — отбранчуйся (логику выбора базы см. в `/feature` шаг 2)
- Если мы уже на нужной ветке — продолжай

### 3. Определи точку входа
По статусу определи последний завершённый шаг и что делать дальше:

| Последний статус | Следующий шаг |
|---|---|
| нет записей | `/spec $ARGUMENTS` |
| `spec-review: done`, нет ветки | `/feature $ARGUMENTS` |
| `spec-review: done`, ветка есть | `/implement $ARGUMENTS` |
| `implement-iterations: N ≥ 3` | показать историю сбоев, спросить как продолжать |
| `code-review: done`, нет коммитов | шаг 4 из `/feature` (коммиты) |
| `build: failed` / `test-run: failed` / `code-review: failed` | `/implement $ARGUMENTS` с контекстом последнего сбоя |

### 4. Доклад
Покажи пользователю:
- Что уже сделано
- Где остановились и почему
- Предлагаемый следующий шаг

Жди подтверждения, затем продолжи.
