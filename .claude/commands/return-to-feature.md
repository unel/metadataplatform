Восстанови контекст и продолжи работу над конкретной задачей.

Аргументы: `$ARGUMENTS` — путь к задаче, например `feature/store-query`

## Шаги

### 1. Восстановление контекста
Прочитай:
- `tasks/$ARGUMENTS/status.md` — что сделано, что упало
- `tasks/$ARGUMENTS/spec.md` — напомни себе цель
- `tasks/$ARGUMENTS/deps.md` — зависимости
- Последние коммиты в текущей ветке через `/git`

### 2. Ветка
Через `/git` проверь существует ли ветка для этой задачи (`feature/$ARGUMENTS`, `bug/$ARGUMENTS` и т.д.):
- Если ветка есть — переключись на неё
- Если ветки нет и мы на `main`/`master` — отбранчуйся (логику выбора базы см. в `/feature` шаг 2)
- Если мы уже на нужной ветке — продолжай

### 3. Определи точку входа

**Алгоритм:**

1. Если `status.md` пуст или не существует — запусти `/spec $ARGUMENTS`, остановись.
2. Иначе пройди шаги сверху вниз и найди **первый** со статусом `failed`, `pending` или `needs-recheck` — это и есть точка входа. Перейди к шагу 3а.
3. Если все шаги `done` — просчитай следующий шаг по флоу (шаг 3б).

**3а. Первый незавершённый шаг:**

| Статус шага | Действие |
|---|---|
| `pending` / `needs-recheck` | выполнить шаг заново (`/spec-review`, `/acceptance-write` и т.д.) |
| `*-review: failed` | см. **Цикл ревью** ниже |
| `implement-iterations: N ≥ 3` | показать историю сбоев, ждать инструкций |

**3б. Все шаги done — следующий по флоу:**

| Ситуация | Следующий шаг |
|---|---|
| `spec-review: done`, нет ветки | `/feature $ARGUMENTS` |
| `spec-review: done`, ветка есть | `/implement $ARGUMENTS` |
| `code-review: done`, нет коммитов | шаг 4 из `/feature` (коммиты) |
| всё включая коммиты — done | сообщить пользователю, задача завершена |

### Цикл ревью после провала

Применяется к любому шагу вида `<step>-review: failed`. Алгоритм:

1. Прочитай `acceptance-iterations` (или аналогичный счётчик) из `tasks/$ARGUMENTS/status.md`. Если нет — считай 0.
2. Прочитай `max_retries` для шага из `.claude/project.yaml`.
3. Если `iterations ≥ max_retries` — **стоп**: показать все провалы, все гипотезы, ждать инструкций от пользователя.
4. Иначе:
   - Инкрементируй счётчик в `status.md`
   - Прочитай `retry_escalation[iterations]` из `.claude/project.yaml` — выбери тон и глубину
   - Вызови соответствующий fix-скилл:

     | Шаг с failed | Fix-скилл |
     |---|---|
     | `spec-review` | `/fix-spec $ARGUMENTS` |
     | `acceptance-review` | `/fix-acceptance $ARGUMENTS` |
     | `test-review` | `/fix-test $ARGUMENTS` |
     | `code-review` | `/fix-code $ARGUMENTS` |
     | `doc-review` | `/fix-doc $ARGUMENTS` |

   - По завершении — **обязательно** запусти соответствующий ревью-шаг снова
   - Если ревью снова failed — повтори цикл с п.1

### 4. Доклад и продолжение
Покажи пользователю:
- Что уже сделано
- Где остановились и почему
- Что делаешь прямо сейчас

Продолжи работу немедленно — не жди подтверждения.
