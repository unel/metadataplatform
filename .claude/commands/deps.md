Анализ зависимостей фичи — находит, фиксирует и обновляет связи с другими фичами.

Аргументы: `$ARGUMENTS` — путь к фиче, например `store/query`

## Подготовка

1. Прочитай `docs/deps/` — список всех папок, каждая папка = тип связи
2. Для каждого типа прочитай `docs/deps/<type>/description.md`
3. Если `docs/deps/` не существует — сообщи об этом и остановись

## Шаги

4. Прочитай `tasks/$ARGUMENTS/deps.md` если существует — это текущие зафиксированные зависимости
5. Проанализируй спеки и код на новые/изменившиеся связи:
   - для каждого типа связи читай соответствующие `find-outgoing-spec.md`, `find-outgoing-code.md`, `find-incoming-spec.md`, `find-incoming-code.md` и применяй инструкции
   - анализируй `tasks/$ARGUMENTS/spec.md` и код фичи (исходящие)
   - анализируй `spec.md` и код других фич (входящие)
6. Покажи diff — новые, устаревшие, изменившиеся связи — жди подтверждения или правок
7. После подтверждения:
   - обнови `tasks/$ARGUMENTS/deps.md`
   - для каждой связи имеющей обратную (`depends-on`↔`used-by`, `extends`↔`extended-by`, `blocks`↔`blocked-by`) — автоматически обнови `deps.md` целевой фичи

## Формат deps.md

```markdown
# deps — store/query

## depends-on
- `store/connection` — использует пул соединений

## used-by
- `api/entities` — читает entity через store
- `spawner/jobs-watch` — поллит jobs

## similar-to
- `spawner/events-watch` — одинаковый паттерн поллинга

## overlaps
- `api/commands` — оба проксируют команды в store
```

Секции без связей — не включать.

## Саб-скиллы

Вызываются с теми же аргументами, делают часть работы:
- `/deps-spec` — только анализ по спекам (шаги 5 только spec файлы)
- `/deps-code` — только анализ по коду (шаги 5 только code файлы)
- `/deps-in` — только входящие связи
- `/deps-out` — только исходящие связи

## Важно

- Не добавлять связи ради галочки — только реальные, подтверждённые анализом
- Если обнаружен `conflicts` — явно предупредить пользователя и предложить запустить `/rfc`
- Обратные связи (`used-by`, `extended-by`, `blocked-by`) проставлять автоматически в целевых фичах
