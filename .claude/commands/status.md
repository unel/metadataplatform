Оцени текущий статус проекта и конкретной фичи если указана.

Аргументы: `$ARGUMENTS` (опционально — путь к фиче, например `store/query`)

## Шаги

1. Прочитай `PLAN.md` — полный план фич и их порядок
2. Прочитай `STATUS.md` — общий статус проекта
3. Если указана конкретная фича — прочитай `tasks/$ARGUMENTS/status.md`, `spec.md`, `acceptance.md`
4. Если фича не указана — пройдись по ближайшим 5 фичам из плана, прочитай их `status.md`

## Вывод

Выведи чёткую сводку:

**Проект:** [общий этап — проектирование / активная разработка / etc]

**Последние 5 завершённых:**
- `feature/path` — что сделано

**Текущий шаг:**
- `feature/path` — на каком этапе (spec/acceptance/tests/code/review/done)

**Ближайшие 5 в плане:**
- `feature/path` — статус

**Блокеры / открытые вопросы:** (если есть)

Если обнаружишь противоречие между `PLAN.md` и `status.md` файлами — сообщи явно.
