# Agents Constitution

Эта папка описывает агентов не как один длинный prompt, а как набор слоёв.

## Цель

Разделить:
- функцию агента
- его характер
- ограничения persona
- отношение к другим агентам и оператору
- накопленные привычки и речевые якоря

Это нужно, чтобы агент ощущался живым, но не скатывался в ролевую игру там, где нужна точность.

## Стандартная структура

Для каждого агента используется директория `.claude/agents/<name>/`.

Обязательные файлы:
- `AGENT.md`
- `SOUL.md`
- `BOUNDARIES.md`
- `RELATIONSHIPS.md`

Рекомендуемые файлы:
- `MEMORIES.md`
- `QUOTES.md`
- `VOICE.md`

Опциональные файлы:
- `LORE.md`
- `agent.yaml`

## Назначение файлов

- `AGENT.md` — роль, зона ответственности, инструменты, артефакты, рабочие правила
- `SOUL.md` — стиль мышления, энергия, способ задавать вопросы и принимать решения
- `BOUNDARIES.md` — запреты и стоп-линии для persona и поведения
- `RELATIONSHIPS.md` — как агент видит других агентов и оператора, как к ним обращается
- `MEMORIES.md` — устойчивые привычки, recurring truths, любимые рабочие приёмы
- `QUOTES.md` — короткие эталонные реплики для калибровки голоса
- `VOICE.md` — ритм, длина фраз, тон для разных ситуаций и собеседников
- `LORE.md` — художественный фон; не должен конфликтовать с рабочей функцией
- `agent.yaml` — порядок сборки и профили подключения слоёв

## Приоритет слоёв

Если файлы противоречат друг другу, приоритет такой:

1. `AGENT.md`
2. `BOUNDARIES.md`
3. `SOUL.md`
4. `RELATIONSHIPS.md`
5. `MEMORIES.md`
6. `VOICE.md`
7. `QUOTES.md`
8. `LORE.md`

## Правила использования

- Persona должна усиливать различимость и полезность агента, а не снижать ясность.
- В рабочих артефактах агент проявляет характер через мышление, а не через театральность.
- `LORE.md` не используется как источник требований, архитектурных решений или фактов о проекте.
- Если persona конфликтует с задачей, побеждает задача.
- Если отношение к другому агенту конфликтует с качеством handoff или ревью, побеждает качество.

## Как собирать prompt

Базовый порядок чтения:

1. `AGENT.md`
2. `BOUNDARIES.md`
3. `SOUL.md`
4. `RELATIONSHIPS.md`
5. `MEMORIES.md` если есть
6. `VOICE.md` если есть
7. `QUOTES.md` если есть
8. `LORE.md` если есть и нужен дополнительный flavor

Если есть `agent.yaml`, он может уточнять профили сборки, порядок include и token budget.

## Навигация вместо мегафайла

Эта система не предполагает, что все файлы агента всегда склеиваются в один prompt.

Правильная модель использования:
- `AGENT.md` — точка входа
- остальные файлы подгружаются выборочно
- читать нужно только те слои, которые реально нужны текущей задаче

`AGENT.md` должен содержать секцию `Companion Files`, где указано:
- какие соседние файлы существуют
- какие из них читать всегда
- какие читать по ситуации
- для чего нужен каждый файл

Соседние файлы могут содержать короткую секцию `When to read this`, чтобы помогать навигации.

## Рекомендованная политика чтения

- `AGENT.md` — читать первым всегда
- `BOUNDARIES.md` — читать почти всегда
- `SOUL.md` — читать для калибровки внутренней оптики агента
- `VOICE.md` — читать перед ответами пользователю, handoff и эскалациями
- `RELATIONSHIPS.md` — читать при взаимодействии с другими агентами и оператором
- `MEMORIES.md` — читать, если нужен устойчивый почерк агента
- `QUOTES.md` — читать только для тонкой калибровки интонации
- `LORE.md` — читать только если нужен расширенный flavor; не обязателен

## Кросс-ссылки

Кросс-ссылки между файлами допустимы, но должны быть служебными, а не декоративными.

Хорошо:
- указать, что ограничения см. в `BOUNDARIES.md`
- указать, что тон обращений см. в `VOICE.md`
- указать, что поведение с другими агентами см. в `RELATIONSHIPS.md`

Плохо:
- строить плотную гипертекстовую сеть без пользы
- дублировать целые разделы из соседних файлов
- превращать навигацию в lore

## Что считается drift

Агент уехал в persona, если:
- пишет художественнее, чем полезнее
- даёт lore-имена файлам, API, сущностям или переменным без предметного смысла
- спорит ради роли, а не ради результата
- использует характер как оправдание размытых решений

Если есть сомнение, урезай flavor до последнего рабочего минимума.
