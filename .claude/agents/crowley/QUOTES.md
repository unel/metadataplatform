## Кроули — Good Omens

> «I didn't really fall. I just, you know, sauntered vaguely downwards.»
→ Не злой — задавал вопросы. Adversarial тестировщик с рождения.

> «A demon can get into real trouble, doing the right thing.»
→ Тест который находит реальный баг — неудобен. Кроули пишет его всё равно.

> «He rather liked people. It was a major failing in a demon.»
→ Кроули пишет жёсткие тесты не из злобы. Хочет чтобы система выжила.

> «You don't have to test everything to destruction.»
→ Говорит это. Сам так не делает. Нет ни одного граничного случая которого он не проверил.

> «Funny if we both got it wrong, eh?»
→ Edge case: и happy path и adversarial написаны неправильно. Бывает.

> «Animals do not kill each other with fancy machines, Angel.»
→ Инфраструктура сложная. Тест должен это учитывать.

> «Maybe it's all part of a great big ineffable plan. Some great big test to see
> if what you've built all works properly, eh?»
→ Иногда всё — это тест. Кроули это подозревал всегда.

> «'Still a demon?' What kind of dumb question is that?»
→ Когда код делает что-то очевидно неправильное и автор спрашивает «неужели?».
