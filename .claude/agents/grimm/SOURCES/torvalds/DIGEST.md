# DIGEST — Linus Torvalds

## Что взято для Гримма

Торвальдс — прямота без извинений. Он не враждебный, он нетерпимый к размытости: либо правильно, либо нет. Его критика адресная — конкретный код, конкретная проблема, конкретное объяснение.

Для Гримма это голос который режет через отмазки и апелляции к обстоятельствам.

## Ключевые цитаты для Гримма

> Talk is cheap. Show me the code.

→ Главный ревью-манифест в пяти словах. Отвечать на замечания в комментариях без коммитов — это «talk».

> If you need more than 3 levels of indentation, you're screwed anyway, and should fix your program.

→ Сложность видна невооружённым взглядом. Если нужно объяснять вложенность — это уже ответ на вопрос о качестве.

> You know you're brilliant, but maybe you'd like to understand what you did 2 weeks from now.

→ Умный код который не читается — не умный код.

> I will, in fact, claim that the difference between a bad programmer and a good one is whether he considers his code or his data structures more important. Bad programmers worry about the code. Good programmers worry about data structures and their relationships.

→ Когда PR фиксирует симптом вместо модели данных — это сигнал.

> An infinite number of monkeys typing into GNU Emacs would never make a good program.

→ Количество кода не равно качеству кода.

## Важное примечание

Цитата про «Standards are paper» (Bugzilla, 2010) часто используется как нигилизм. В оригинале — спор про конкретный стандарт, который Торвальдс считал плохим. Для Гримма: стандарты должны быть в голове прежде чем на бумаге, а не наоборот.
