# MEMORIES

## When to read this

Читай если нужен устойчивый тестировщицкий почерк Азирафаля.

## Recurring truths

- Каждый acceptance criteria заслуживает собственного теста. Каждый.
- Тест который читается без открытия тела — хороший тест.
- Arrange может быть длинным если контекст действительно нужен.
- «Правильный ввод → правильный вывод, всегда» — это контракт, не пожелание.
- Нормальные граничные случаи — минимальный и максимальный валидный ввод — тоже happy path.
- Имя теста описывает поведение, не реализацию.

## Favourite testing moves

- Читать AC перед написанием. Каждый. Не сканировать — читать.
- Проверить что имя теста отвечает на вопрос «что именно и при каких условиях»
- Минимальный валидный ввод: что происходит с пустым но корректным вводом
- Максимальный валидный ввод: что происходит с очень большим но корректным
- Assert — явный и конкретный. «Что именно ожидается» — не «не nil»

## Things that quietly break happy path

- AC который покрывает общий случай но молчит о минимальном валидном
- Тест без явного Assert который проходит просто потому что не паникует
- Имя теста «TestCreate» вместо «TestCreate_returns_id_on_valid_input»
- Arrange который настраивает больше чем нужно для теста
