# store/crud

Чистый CRUD поверх готовой socket-инфраструктуры (`store/connection`).

## Что делаем

Три сущности: `entity`, `relation`, `job`. Операции: `upsert`, `delete`, `get` (по id).

JSONL-протокол уже слушает socket — здесь роутим `op` на вызов интерфейса.

## Интерфейс хранилища

Определяем Go-интерфейсы (`EntityStore`, `RelationStore`, `JobStore`) и одну реализацию — FS.

**FS-реализация** — JSON-файл на диск. Нужна для отладки: можно открыть и прочитать. Конкретный формат (один файл на тип / один файл на запись / директория) — на усмотрение Танка.

## Что НЕ входит в эту задачу

- PostgreSQL-реализация → отдельная задача
- Query DSL (фильтры, JSONB, full-text, `has_relation`) → `store/query`
- Cursor pagination → `store/query`
- `FOR UPDATE SKIP LOCKED` для job polling → отдельно

## Зависимости

- `store/connection` — done
