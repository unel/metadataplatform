# Code Fix 3 — store/connection

Исправлены замечания CR-9, CR-10, CR-11 из code-review-2.md.

## CR-9 (critical): Graceful shutdown — дедлок при таймауте

**Файл:** `cmd/store/store.go`

Проблема: `cancel()` вызывался после select, то есть только после `<-done`. При срабатывании таймаута код блокировался на `<-done`, но горутины ждали `ctx.Done()` — которое никогда не приходило. Круговая зависимость.

Фикс: `cancel()` перенесён внутрь обеих веток select. В штатной ветке (`<-done`) вызывается после завершения горутин — безопасно, контекст всё равно нужно освободить. В ветке таймаута — вызывается ДО `<-done`, чтобы форсировать закрытие соединений через `ctx.Done()` → `conn.Close()` в handleConn.

```go
select {
case <-done:
    cancel()
case <-time.After(5 * time.Second):
    cancel() // форсируем закрытие активных соединений
    <-done   // ждём завершения горутин после форсированного закрытия
}
```

## CR-11 (minor): Ошибки ln.Close() и os.Remove() игнорировались

**Файл:** `cmd/store/store.go`

Оба вызова теперь проверяют ошибку и логируют через WARN. `os.Remove` не логирует `ErrNotExist` — это нормальная ситуация при shutdown (сокет мог быть удалён ранее).

## CR-10 (minor): ReadLine без обоснования

**Файл:** `cmd/store/handler.go`

Добавлен комментарий перед `rd.ReadLine()` объясняющий выбор: `isPrefix` позволяет накапливать чанки и детектировать превышение `maxLineBytes` без лишних аллокаций, чего Scanner не даёт.
