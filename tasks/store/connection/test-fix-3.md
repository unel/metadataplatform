# Test Fix 3 — store/connection

## Причина фикса

Дедлок в `TestHighThroughput_1000Lines_NoneDropped`.

Тест посылал 1000 строк синхронно в цикле, не читая ответы. Store отвечает эхом синхронно на каждую строку. Unix socket buffer конечен (~208 КБ для loopback). После заполнения буфера:
- тест блокируется на `fmt.Fprintf` (write syscall)
- store блокируется на записи эха (write syscall)

Оба ждут друг друга. Дедлок. 10 секунд таймаут, `t.Fatal`.

## Что исправлено

**Файл:** `cmd/store/connection_adversarial_test.go`
**Функция:** `TestHighThroughput_1000Lines_NoneDropped`

Запись вынесена в горутину. Чтение идёт в основной горутине параллельно с записью. Буфер не забивается — store пишет эхо, тест его немедленно читает, запись продолжается.

Канал `writeErr` (буферизованный, размер 1) собирает ошибку из горутины. После получения всех `n` ответов — `require.NoError(t, <-writeErr)` проверяет что запись прошла без ошибок.

## Что не изменилось

Сценарий тот же: 1000 строк, ни одна не потеряна. Assertion тот же. Добавлена только горутина для записи и проверка её ошибки.

## Статус

Дедлок устранён. Новых тестов не добавлено — сценарий уже был в acceptance.md (пропускная способность).
