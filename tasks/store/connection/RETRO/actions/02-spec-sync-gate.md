# Action: Spec sync gate

## Проблема

`MaxConnections` добавлен через CR-5 (code-review) как требование стандарта проекта. В `spec.md` и `acceptance.md` поле не появилось. Финальный код имеет лимит соединений, спека — нет. Реализация использует `bufio.Reader.ReadLine`, спека описывает `bufio.Scanner` / `ErrTooLong`. Следующий читатель спеки получит неверную картину о том как работает фича.

## Решение

Ввести правило "Spec sync gate": если в итерации code-review появляется новое поле конфига, новый параметр или изменяется поведение — обновление `spec.md` и `acceptance.md` обязательно в той же итерации. Без этого ревью не считается закрытым.

Конкретно:
- В скилл `code-review` добавить пункт: "если замечание вводит новое поле или меняет поведение → выдать задачу на обновление spec.md + acceptance.md в том же code-fix"
- Статус code-review не может быть `pass` если конфиг/поведение изменилось а spec не обновлена
