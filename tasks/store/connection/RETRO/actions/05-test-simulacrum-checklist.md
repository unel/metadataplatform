# Action: пункт про тест-симулякры в test-review

## Проблема

`TestStore_GracefulShutdown_ForcesCloseAfter5Seconds` использовал `sp.Stop()` со встроенным 6-секундным таймаутом. Тест завершался по внешней обёртке, а не по поведению store. Внутри был deadlock (CR-9). Тест зелёный, баг живой — прошёл через test-write и два раунда test-review незамеченным. NFT-06 в первой версии был нефальсифицируемым: если store завершался мгновенно, тест проходил.

## Решение

Добавить в чек-лист `test-review` два пункта:

**Пункт 1 — таймаут-обёртки:**
"Если тест содержит таймаут-обёртку — проверь что тест завершается именно по тому условию которое проверяет, а не по таймауту обёртки. Сигнал: если внешний таймаут ≥ ожидаемому внутреннему таймауту — тест скорее всего не фальсифицируем."

**Пункт 2 — фальсифицируемость:**
"Может ли этот тест провалиться если система ничего не делает? Если нет — тест нефальсифицируем. Для тестов на негативное поведение и таймауты это обязательная проверка."
