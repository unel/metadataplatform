# Action: чек-лист code-review Go — все файлы в каждом раунде

## Проблема

Три файла в фиче store/connection: `store.go`, `handler.go`, `main.go`. Первые два проверялись дважды, `main.go` — один раз из трёх раундов ревью. CR-12 (паника при двойном SIGTERM через двойной вызов `signal.Stop`) существовал с первой версии, найден только в третьем раунде. Воспроизводится любым нетерпеливым оператором. Не экзотика.

## Решение

Добавить в чек-лист code-review Go-сервисов:

```
Обязательно в каждом раунде:
- [ ] Проверены все файлы пакета, включая main.go
- [ ] Signal handling: регистрация сигналов, идемпотентность Stop, double-signal поведение
- [ ] Инициализация: порядок, ошибки, cleanup при неудаче
- [ ] При изменении горутинной структуры — обязательный повторный проход всего shutdown path
```

Правило: если в предыдущем раунде изменялась goroutine структура (добавлена/удалена горутина, изменён select, изменён WaitGroup) — следующий раунд ревью начинается с полного прохода shutdown path как отдельного шага.
