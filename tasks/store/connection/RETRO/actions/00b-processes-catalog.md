# Каталог процессов: структура и артефакты

## Структура директорий

```
tasks/<feature>/
  TASK.md                    ← описание задачи: что, зачем, ссылка на план
  processes/
    00-research/
      01-interview/
      02-web/
    01-spec/
      01-write/
      02-review/
      03-fix/
    02-acceptance/
      01-write/
      02-review/
      03-fix/
    03-tests/
      01-write/
      02-review/
      03-fix/
      04-run/
    04-code/
      01-write/
      02-review/
      03-fix/
    05-docs/
      01-write/
      02-review/
      03-fix/
    06-retro/
      01-recall/
      02-discussion-1/
      03-grouping/
      04-discussion-2/
      05-abstraction/
      06-discussion-3/
      07-action-points/
      08-discussion-4/
      09-final/
```

Код, тесты, документация — **не в tasks**, живут в реальной структуре проекта (`cmd/`, `docs/` и т.д.).

---

## Мета-артефакты каждого процесса

Каждая директория процесса содержит:

```
<process>/
  status-log.md              ← хронология прогонов, актуальный статус — последняя строка
  plan.md                    ← шаги, порядок, исполнители
  checklist.md               ← базовый чек-лист + подключённые расширения
  notes-<агент>.md           ← заметки участника (один файл на агента)
  complaints-<агент>.md      ← жалобы участника (один файл на агента)
  report-001.md              ← отчёт первого прогона
  report-002.md              ← отчёт второго прогона (если был)
  ...
```

`report-NNN.md` — основной артефакт прогона. Для ревью это замечания, для фикса — что изменили, для run — результаты тестов.

## Входные и выходные артефакты по группам

### 00-research

**01-interview**
| | |
|---|---|
| **Вход** | `TASK.md`, `PROJECT.md`, `PLAN.md` |
| **Выход** | `report-001.md` (выжимка из разговора: что хочет пользователь, зачем, контекст) |
| **Исполнитель** | Танк |
| **Примечание** | Разговор с пользователем — не угадывать намерения, уточнять явно |

**02-web**
| | |
|---|---|
| **Вход** | `00-research/01-interview/report-*.md`, `TASK.md`, `PROJECT.md` |
| **Выход** | `report-001.md` (похожие решения, паттерны, failure modes — с URL источников inline) |
| **Доп. файлы** | `searches.md` (что гуглил), `sources.md` (URL источников с кратким описанием) — если данных много |
| **Исполнитель** | Бо |
| **Примечание** | Каждый факт в report-001.md сопровождается ссылкой на источник. URL фигурируют и в report, и в sources.md — чтобы можно было перепроверить |

### 01-spec

**01-write**
| | |
|---|---|
| **Вход** | `TASK.md`, `PROJECT.md`, `PLAN.md`, `00-research/01-interview/report-*.md`, `00-research/02-web/report-*.md` |
| **Выход** | `report-001.md` (содержит spec.md) |
| **Исполнитель** | Танк |

**02-review**
| | |
|---|---|
| **Вход** | spec из `01-write/report-*.md`, `PROJECT.md` |
| **Выход** | `report-NNN.md` (замечания) |
| **Исполнитель** | Гримм |
| **Примечание** | При `failed` → откат к `01-write` или `03-fix` |

**03-fix**
| | |
|---|---|
| **Вход** | spec, `02-review/report-NNN.md` |
| **Выход** | `report-NNN.md` (обновлённая spec + что изменили) |
| **Исполнитель** | Танк |

### 03-tests

**01-write**
| | |
|---|---|
| **Вход** | spec, acceptance |
| **Выход** | `report-001.md` (ссылки на тест-файлы в `cmd/`) |
| **Исполнители** | Кроули (adversarial), Азирафаль (happy path) |
| **Примечание** | testhelpers пишутся первыми — они фундамент |

**02-review**
| | |
|---|---|
| **Вход** | тест-файлы, testhelpers, spec, acceptance |
| **Выход** | `report-NNN.md` (замечания) |
| **Исполнитель** | Гримм |
| **Примечание** | Проверяет testhelpers первыми |

**03-fix**
| | |
|---|---|
| **Вход** | тест-файлы, `02-review/report-NNN.md` |
| **Выход** | `report-NNN.md` (что изменили, grep по паттерну выполнен) |
| **Исполнители** | Кроули, Азирафаль |

**04-run**
| | |
|---|---|
| **Вход** | тест-файлы, код реализации |
| **Выход** | `report-NNN.md` (результаты `go test`) |
| **Исполнитель** | Харли |
| **Примечание** | При `failed` → `needs-clarification` или откат к `03-fix` / `04-code/03-fix` |

### 06-retro

**01-recall**
| | |
|---|---|
| **Вход** | все артефакты фичи, существующие `notes-*.md`, `complaints-*.md` |
| **Выход** | `report-001.md` (сводка: что обновлено, сколько наблюдений добавил каждый) |
| **Исполнители** | все участники параллельно |

**02-discussion-1** (первичные выжимки)
| | |
|---|---|
| **Вход** | `notes-*.md`, `complaints-*.md`, `notes-user.md`, `complaints-user.md` |
| **Выход** | `report-001.md` (выжимки от каждого участника) |
| **Исполнители** | все тиммейты → Харли собирает |

**03-grouping**
| | |
|---|---|
| **Вход** | выжимки из `02-discussion-1` |
| **Выход** | `report-001.md` (проблемы по категориям: требования / процессные / инструментальные / командные) |
| **Исполнитель** | Харли |

**04-discussion-2** (уточнение)
| | |
|---|---|
| **Вход** | `03-grouping/report-001.md` |
| **Выход** | `report-001.md` (уточнённые проблемы с конкретными примерами) |
| **Исполнители** | Харли задаёт вопросы → тиммейты отвечают |

**05-abstraction**
| | |
|---|---|
| **Вход** | `04-discussion-2/report-001.md` |
| **Выход** | `report-001.md` (3-5 корневых причин поверх симптомов) |
| **Исполнитель** | Харли |

**06-discussion-3** (проверка абстракций)
| | |
|---|---|
| **Вход** | `05-abstraction/report-001.md` |
| **Выход** | `report-001.md` (подтверждённые/скорректированные абстракции) |
| **Исполнители** | Харли + тиммейты + пользователь |

**07-action-points**
| | |
|---|---|
| **Вход** | `06-discussion-3/report-001.md` |
| **Выход** | `report-001.md` (список action points), `actions/*.md` (по одному файлу на action point) |
| **Исполнитель** | Харли |

**08-discussion-4** (проверка action points)
| | |
|---|---|
| **Вход** | `07-action-points/actions/*.md` |
| **Выход** | `report-001.md` (скорректированные action points) |
| **Исполнители** | Харли + тиммейты + пользователь |

**09-final**
| | |
|---|---|
| **Вход** | все результаты `06-retro/` |
