# Action: fix-report обязателен после каждого failed-ревью

## Проблема

14 шагов в status.md по фиче store/connection, ни одного `*-fix-report.md`. Ревью находили баги, артефакты переписывались — следов нет. Следующая фича начнёт с теми же слепыми пятнами, потому что нет документированной памяти о том что именно было проблемой и как это решили. История итераций потеряна.

## Решение

Fix-report обязателен после каждого failed-ревью. Без него status.md нельзя двигать в `done` после фикса.

Формат `<artifact>-fix-report-N.md`:
```
# Fix report: <артефакт>, итерация N

## Замечания из ревью

| ID | Замечание | Критичность | Что изменил |
|---|---|---|---|
| CR-1 | ... | critical | ... |

## Что НЕ менялось и почему

- ...

## Открытые вопросы после фикса

- ...
```

Правило: агент который выполняет фикс пишет fix-report. Агент который делает ревью — только ревью-файл, не fix-report.
