# Action: RFC — Харли и доступ к Agent tool

## Проблема

Харли как субагент не имеет доступа к `Agent` tool. Оркестрация требует ручного вмешательства оператора на каждом шаге. Recall по store/connection занял несколько итераций вместо одной. Флоу не двигается автоматически. Это системное ограничение платформы — субагенты не могут спаунить других субагентов.

Цитата из complaints-харли.md: "Меня запустили как субагента для оркестрации других субагентов — но Agent tool у субагента недоступен. Это как попросить дирижёра взмахнуть палочкой, но не дать ему рук."

## Что нужно решить

1. Это ограничение платформы или конфигурации? Можно ли дать субагенту Agent tool?
2. Если нет — как переработать скиллы чтобы они были корректны при выполнении как main thread (через `_Режим: Харли Куин_`) и не ломались когда оркестратор — субагент?
3. Альтернатива: все оркестрационные скиллы всегда выполняются только как main thread, никогда как субагент. Харли-субагент получает урезанный набор задач.

## Шаги

- [ ] Исследовать возможности платформы: может ли субагент получить Agent tool через конфиг?
- [ ] Если нет — написать RFC с предложением архитектурного изменения
- [ ] Обновить `.claude/commands/` — явно указать для каждого скилла "выполняется только как main thread" или "может быть субагентом"
