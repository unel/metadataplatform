# Action: стандарт slog в тестах + grep-конвенция при фиксе паттерна

## Проблема

`waitForMsg("conn_id")` всегда возвращал `false` — `slog` пишет `conn_id` в `attrs`, не в `msg`. Функция работала как `time.Sleep` с лишним синтаксисом. После первого ревью исправили в одном месте, не сделали grep — тот же баг нашли в другом тесте в следующем раунде. Нет стандарта про структуру slog-вывода в тестах. Нет конвенции "grep по паттерну перед закрытием замечания".

## Решение

### Часть 1: стандарт slog в тестах

Добавить в `docs/standards/architecture/tdd/` параграф:

Структурированный лог имеет два слоя:
- `msg` — человекочитаемый шаблон (может меняться, локализоваться, потерять поля без смены семантики)
- `attrs` — структурированные поля (это контракт)

Тесты должны проверять `attrs`, не подстроку в `msg`. Пример:

```go
// Неправильно:
waitForMsg("conn_id")  // ищет в msg — всегда false

// Правильно:
waitForAttr("conn_id", expectedID)  // ищет в attrs
```

Формулировать через принцип (не "это slog/attrs"), чтобы правило работало при смене логгера.

### Часть 2: grep-конвенция

В чек-лист code-fix и test-fix: "если ревью указало на паттерн — сформулируй паттерн словами (не буквальную строку), затем grep по файлу/пакету и убедись что других экземпляров нет перед закрытием замечания".
