# code-fix-4: CR-12 — паника при двойном сигнале

## Замечание

Канал `sig` буферизован на 1. После первого Ctrl+C горутина закрывала `stopCh` и завершалась.
Если второй сигнал успевал попасть в буфер — следующий reader канала (если бы был) получил бы его, но
реальная проблема: горутина читает канал в цикле без защиты. При наличии второго сигнала в буфере
возможен повторный `close(stopCh)` → паника `close of closed channel`.

## Исправление

**Файл:** `cmd/store/main.go`

Добавлен вызов `signal.Stop(sig)` сразу после приёма первого сигнала, до `close(stopCh)`.
`signal.Stop` отменяет регистрацию канала — OS больше не будет доставлять сигналы в `sig`,
второй Ctrl+C не попадёт в канал.

```go
// было
go func() {
    <-sig
    close(stopCh)
}()

// стало
go func() {
    <-sig
    signal.Stop(sig)
    close(stopCh)
}()
```
