# Как следить за обновлениями

## Источники

- **bun.sh/blog** — анонсы версий, детальные разборы новых возможностей
- **github.com/oven-sh/bun/releases** — changelog
- **twitter/x: @jarredsumner** — автор Bun, анонсирует изменения

## Что искать

- Новые встроенные API (Bun часто добавляет то что раньше требовало npm-пакетов)
- Изменения в производительности — иногда меняется рекомендуемый способ делать что-то
- Новые возможности test runner'а
- Изменения совместимости с Node.js API

## Паттерн для устаревших паттернов

```typescript
// OUTDATED(Bun 1.0): использует node:crypto вместо встроенного Bun.hash
// актуальный подход: Bun.hash() или Bun.CryptoHasher
// рефакторинг: tasks/feature/bun-native-crypto
import { createHash } from 'node:crypto'
```

## При обновлении Bun

1. Читай changelog — Bun иногда меняет поведение между версиями
2. Проверяй совместимость с используемыми npm-пакетами
3. Ищи OUTDATED комментарии
