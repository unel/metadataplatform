# Базовый чек-лист: 04-code-fix

## Контекст

- [ ] Review report прочитан — список замечаний с классификацией
- [ ] Затронутые файлы прочитаны

## Red run (воспроизведение бага)

- [ ] Тест воспроизводящий баг написан или найден
- [ ] Тест падает до фикса — `verify: Red run пройден`

## Перед фиксом

- [ ] План фикса согласован с пользователем

## Фикс

- [ ] Исправлены только замечания из review — нет попутного рефакторинга
- [ ] Приоритет соблюдён: сначала critical, потом warning, потом note
- [ ] Каждый модуль ≤ 150 строк (без комментариев)

## Невозможные фиксы

- [ ] Замечания нельзя исправить без изменения acceptance — задокументированы с объяснением
- [ ] `clarification` записан, оркестратор уведомлён если применимо

## Тесты после фикса

- [ ] Все тесты зелёные после исправления — `verify: Green run`

## Grep-конвенция

- [ ] Для каждого паттерна из замечаний — grep по файлу/пакету, все экземпляры найдены и исправлены

## Fix-report

- [ ] Fix-report написан по шаблону `docs/standards/templates/fix-report.md`
- [ ] Статус не выставлен в `done` без fix-report (если ревью было `failed`)

## Отчёт

- [ ] Список исправленных замечаний с описанием что сделано
- [ ] Список неисправленных с объяснением

## Движение дальше

- [ ] Все исправимые замечания закрыты → report написан, возврат к 04-code/02-review
- [ ] Если требуется изменение acceptance → `clarification`, откат к 02-acceptance/03-fix
