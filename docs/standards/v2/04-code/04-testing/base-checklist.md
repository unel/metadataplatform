# Базовый чек-лист: 04-code-testing

## Контекст

- [ ] Code-review прошёл — последний report из `04-code/02-review/` или `04-code/03-fix/` со статусом `done`
- [ ] Команда запуска тестов известна (из `PROJECT.md`)

## Запуск

- [ ] Запущены все тесты: `tests/happy/` и `tests/adversarial/`
- [ ] Запущено с детальным выводом (`-v` или аналог)
- [ ] Запущено с `-race` если поддерживается

## Результат

- [ ] **Все тесты зелёные** — это обязательный результат, не опциональный

## Интерпретация падений (если есть)

- [ ] Для каждого упавшего теста определён источник: код / тест / acceptance
- [ ] Не менялись тесты чтобы пройти — менялся только код
- [ ] Не угадывалось правильное поведение при неясном acceptance — эскалировано

## Отчёт

- [ ] Итог: прошло / упало / пропущено
- [ ] Для каждого падения: тест → причина → что делать

## Движение дальше

- [ ] Все тесты зелёные → report `done`, следующий 05-docs/01-write
- [ ] Тест падает из-за ошибки в коде → report `failed`, следующий 04-code/03-fix
- [ ] Тест падает из-за ошибки в тесте → report `failed`, следующий 03-tests/03-fix
- [ ] Тест падает из-за проблемы в acceptance → `clarification`, откат к 02-acceptance/03-fix
