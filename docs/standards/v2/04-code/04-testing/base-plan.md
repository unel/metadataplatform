# Базовый план: 04-code-testing

## Контекст

Это **TDD Green** шаг. Реализация написана, code-review пройдено. Ожидаемый результат: все тесты зелёные.

Отличие от `03-tests/04-run` (Red): там тесты должны были упасть. Здесь — обязаны пройти.

## 1. Сбор контекста

- Проверь что code-review прошёл: последний `04-code/02-review/report-*.md` или `04-code/03-fix/report-*.md` — статус `done`
- Прочитай `PROJECT.md` — команда для запуска тестов

## 2. Запуск

- Запусти полный suite: `tests/happy/` и `tests/adversarial/`
- С детальным выводом (-v или аналог)
- С race detector если поддерживается (-race в Go)

## 3. Интерпретация результатов

**Ожидаемый результат: все зелёные.** Если тест упал — это проблема, не норма.

Для каждого упавшего теста определи источник:

- **Ошибка в коде** — код не реализует то что требует тест → откат к `04-code/03-fix`
- **Ошибка в тесте** — тест написан неверно (но acceptance корректен) → откат к `03-tests/03-fix`
- **Проблема в acceptance** — acceptance описывает недостижимое или противоречивое → `clarification`, откат к `02-acceptance/03-fix`

Не меняй тесты чтобы они прошли — меняй код. Не угадывай правильное поведение при неясном acceptance — эскалируй.

## 4. Формирование отчёта

- Итог: прошло / упало / пропущено
- Для каждого падения: тест → причина → что делать

## Движение вперёд и назад

**Done когда:** все тесты зелёные, report написан
**Следующий:** 05-docs/01-write
**Передаём:** report-NNN.md

**Возврат назад:**
- Тест падает из-за ошибки в коде → `failed`, откат к 04-code/03-fix
- Тест падает из-за ошибки в тесте → `failed`, откат к 03-tests/03-fix
- Тест падает из-за проблемы в acceptance → `clarification`, откат к 02-acceptance/03-fix
