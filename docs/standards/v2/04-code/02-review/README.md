# 04-code/02-review: Ревью кода

## Что делает

Гримм проверяет качество реализации: соответствие spec/acceptance, SOLID, читаемость, безопасность, размер модулей. Классифицирует каждое замечание.

## Входящие артефакты

- Код реализации
- `04-code/01-write/report-NNN.md`
- `01-spec/*/report-NNN.md` и `02-acceptance/*/report-NNN.md` — для проверки соответствия

## Исходящие артефакты

- `04-code/02-review/report-NNN.md` — замечания с классификацией

## Навигация

| Условие | Следующий шаг |
|---|---|
| Нет критичных замечаний | 03-tests/04-run (финальный прогон) |
| Есть замечания | 04-code/03-fix |
| Код не соответствует acceptance | `clarification` → уточнение с оркестратором |

## Артефакты процесса

- `base-plan.md` — инструкции для Гримма
- `base-checklist.md` — чек-лист ревью
- `report-NNN.md` — результат (иммутабельный)
