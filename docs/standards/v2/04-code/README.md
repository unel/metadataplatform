# 04-code: Код (Green)

## Что делает группа

TDD Green фаза. Ада пишет минимальный код чтобы тесты прошли, Гримм ревьюит (именование, SLAP, SOLID, OWASP), Ада фиксит. Финальный шаг — `04-testing`: запуск всего suite, все тесты должны пройти.

## Ключевые правила

- Минимально необходимый код — не больше чем нужно для Green
- ≤ 150 строк на модуль (без комментариев)
- SOLID, OWASP Top 10, именование по Martin Clean Code ch.2
- SLAP/Stepdown Rule: один уровень абстракции на метод
- Никогда не менять тесты чтобы они прошли
- Валидация только на системных границах (user input, внешние API)

## Процессы

| Процесс | Исполнитель | Что делает |
|---|---|---|
| `01-write` | Ада | Пишет код — минимальный для Green |
| `02-review` | Гримм | Ревьюит: critical / warning / Nit: |
| `03-fix` | Ада | Фиксит по замечаниям Гримма |
| `04-testing` | Азирафаль | Green run — все тесты должны пройти |

Цикл `02-review → 03-fix` повторяется до `pass`.

## Навигация

| Условие | Следующий шаг |
|---|---|
| `04-testing` — все тесты прошли | `05-docs/01-write` |
| `04-testing` — есть падения | Назад в `04-code/01-write` |
| `02-review` вернул `fail` | `04-code/03-fix` |
