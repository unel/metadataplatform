# Базовый чек-лист: 05-docs-review

## Контекст

- [ ] Документационные файлы прочитаны
- [ ] Код реализации прочитан для проверки точности
- [ ] Spec и acceptance прочитаны для проверки полноты

## Точность

- [ ] Примеры рабочие — соответствуют реальному коду
- [ ] Коды ошибок, форматы ответов — соответствуют реализации
- [ ] Нет утверждений противоречащих коду

## Типы документации (Diataxis)

- [ ] How-to не объясняет "почему так" — только шаги
- [ ] Reference нейтральный — нет советов и мнений
- [ ] Explanation / ADR: поле Context есть и содержит реальные constraints
- [ ] Типы не смешаны в одном документе

## Полнота

- [ ] Описано что делает фича
- [ ] Описано как использовать — с примерами
- [ ] Описано поведение при ошибках
- [ ] Нетривиальные технические решения задокументированы (или явно отсутствуют)
- [ ] API endpoint'ы задокументированы если применимо

## Читаемость

- [ ] Для читателя, не для автора
- [ ] Конкретные примеры есть
- [ ] Нет документирования очевидного
- [ ] Описывает текущее состояние

## Классификация

- [ ] Каждое замечание классифицировано: critical / warning / Nit:
- [ ] Не правил документацию самостоятельно — только описал проблемы

## Движение дальше

- [ ] Нет critical и warning → report `done`, следующий 06-retro/01-recall
- [ ] Есть critical или warning → report `failed`, следующий 05-docs/03-fix
