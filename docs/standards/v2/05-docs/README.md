# 05-docs: Документация

## Что делает группа

Танк пишет документацию по готовому коду, Гримм ревьюит на точность и соответствие Diataxis, Танк фиксит. Выходной артефакт: документация прошедшая ревью.

## Ключевые правила (Diataxis)

Четыре типа документов — не смешивать в одном файле:

| Тип | Когда | Акцент |
|---|---|---|
| Tutorial | Обучение нового пользователя | Action — делаем вместе |
| How-to | Решение конкретной задачи | Action — только шаги, без "почему" |
| Reference | Справочник API/параметров | Нейтральный, без советов |
| Explanation / ADR | Объяснение решения | Cognition — контекст и trade-offs |

- Некорректная документация хуже отсутствия документации
- ADR — append-only, поле Context важнее всего

## Процессы

| Процесс | Исполнитель | Что делает |
|---|---|---|
| `01-write` | Танк | Пишет документацию по коду и spec |
| `02-review` | Гримм | Ревьюит на точность, Diataxis, полноту |
| `03-fix` | Танк | Фиксит по замечаниям Гримма |

## Навигация

| Условие | Следующий шаг |
|---|---|
| `02-review` вернул `pass` | `06-retro/01-recall` |
| `02-review` вернул `fail` | `05-docs/03-fix` |
