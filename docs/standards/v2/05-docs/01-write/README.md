# 05-docs/01-write: Написание документации

## Что делает

Танк пишет документацию к завершённой фиче: что это, как использовать, технические решения которые неочевидны из кода.

## Входящие артефакты

- `04-code/04-testing/report-NNN.md` со статусом `done` — код финализирован
- `01-spec/*/report-NNN.md` — для понимания что описывать
- `02-acceptance/*/report-NNN.md` — для описания поведения
- Код реализации — для технических деталей

## Исходящие артефакты

- Документационные файлы (README.md, TECH.md или обновление существующих)
- `05-docs/01-write/report-NNN.md`

## Навигация

| Условие | Следующий шаг |
|---|---|
| Документация написана | 05-docs/02-review |
| Обнаружено расхождение кода со spec | `clarification` → оркестратор |

## Артефакты процесса

- `base-plan.md` — инструкции для Танка
- `base-checklist.md` — чек-лист качества
- `report-NNN.md` — результат (иммутабельный)
