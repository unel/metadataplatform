# 06-retro: Ретроспектива

## Что делает группа

Харли оркестрирует ретро в пять шагов: агенты вспоминают наблюдения (recall), команда собирается и даёт выжимки (collect), проблемы группируются и докапываются до root cause (analyze), вырабатываются решения с DRI и дедлайном (solve), финальный RETRO.md записывается (write).

## Ключевые правила

- **Prime Directive** (Norm Kerth): атрибуция системе и процессу, не агенту
- Каждое ретро начинается с review предыдущего RETRO.md — устаревшие items списать явно
- **≤ 3 action items**: DRI + конкретное действие + дедлайн; "команда сделает X" не работает
- Критика вещей вне контроля команды → открытые вопросы, не action items

## Процессы

| Процесс | Исполнитель | Что делает |
|---|---|---|
| `01-recall` | Харли + все агенты | Заполняют `notes-<агент>.md` и `complaints-<агент>.md` |
| `02-collect` | Харли | Создаёт команду, собирает выжимки (3–5 пунктов от каждого) |
| `03-analyze` | Харли | Группирует по категориям, 5 Whys, уточняет у тиммейтов |
| `04-solve` | Харли | Вырабатывает решения, верифицирует с командой, закрывает команду |
| `05-write` | Харли | Пишет `RETRO.md` |

## Структура артефактов

```
tasks/<feature>/
  notes-<агент>.md       ← теги: rework / friction / miss / propose / doc
  complaints-<агент>.md  ← сырые жалобы без фильтра
  notes-user.md          ← наблюдения пользователя (если есть)
  complaints-user.md     ← жалобы пользователя (если есть)
  RETRO.md               ← финальный артефакт
```

## Категории проблем (03-analyze)

- **Процессные** — оркестрация, переходы, ретраи
- **Инструментальные** — ограничения платформы/инструментов
- **Требования** — нечёткость, пропущенные сценарии
- **Командные** — взаимодействие агентов, передача контекста
