# Базовый план: 06-retro-solve

## 1. Сбор входных данных

Прочитай `06-retro/03-analyze/report-*.md` — сгруппированные проблемы с деталями и team name.

## 2. Выработка решений

**Ограничение**: не более 3 action items. Если решений больше — приоритизируй: какие дают наибольший эффект при наименьших усилиях. Остальные → открытые вопросы.

**Формат каждого action item** — три обязательных поля:
- **Что**: конкретное действие ("добавить проверку X в чеклист", "разбить шаг Y на два", "обновить стандарт Z")
- **Кто** (DRI — Directly Responsible Individual): конкретный агент или роль, не "команда"
- **Когда**: дедлайн или условие (следующая фича, следующий шаг, конкретная дата)

Для каждой категории проблем предложи конкретное решение:

- **Процессные** → изменение шагов флоу, добавление/удаление переходов, изменение условий триггера
- **Инструментальные** → новые инструменты, конфигурация, workaround или отложенная задача
- **Требования** → изменение стандартов спеки/acceptance, checklist-пункты
- **Командные** → изменение протоколов передачи контекста, формат артефактов

Каждое решение: конкретное, actionable, с приоритетом (высокий / средний / низкий).

## 3. Проверка с тиммейтами

По каждой категории отправь вопрос через SendMessage:

> «Для [категория]: предлагаю [решение]. Видите подводные камни? Что упустила?»

Дождись ответов. Скорректируй решения по обратной связи.

## 4. Завершение команды

После финализации решений:

1. Отправь каждому тиммейту через SendMessage: `message: {type: "shutdown_request"}`
2. Дождись завершения всех тиммейтов
3. Вызови TeamDelete
