# Базовый план: 06-retro-write

## 1. Сбор контекста

- Прочитай `06-retro/04-solve/report-*.md` — таблица решений и actionable backlog

## 2. Запись RETRO.md

Запиши `tasks/<feature>/RETRO.md`:

```markdown
# Ретро: <feature>

_Дата: <дата>_
_Участники: <список агентов>_

## Что шло хорошо

- ...

## Проблемы

### <Категория>

- **<Проблема>** — источник: <агент>
  <Детали из обсуждения>

## Предлагаемые решения

| Проблема | Решение | Приоритет |
|---|---|---|
| ... | ... | высокий / средний / низкий |

## Action items (≤ 3)

| Что | Кто (DRI) | Когда |
|---|---|---|
| ... | <агент/роль> | <следующая фича / дата> |

## Открытые вопросы

- ...
```

## 3. Формирование отчёта

- Ссылка на RETRO.md
- Сводка: N проблем по категориям, N решений, N action items

## Движение вперёд и назад

**Done когда:** RETRO.md написан, report написан
**Следующий:** фича завершена ✓
