# Базовый план: 02-acceptance-review

## 1. Сбор контекста

- Прочитай актуальную spec (последний report из `01-spec/`)
- Прочитай acceptance из последнего `02-acceptance/01-write/report-*.md`

## 2. Проверка покрытия

- Каждый happy path из ФТ имеет хотя бы один сценарий
- Каждый failure mode из ФТ покрыт
- Граничные условия покрыты
- Поведение при недоступности зависимостей покрыто
- НФТ проверяемые тестом — покрыты

## 3. Проверка качества

- Сценарии конкретны: конкретные значения, не "valid input" / "system works"
- **Given** содержит только контекст — не действия
- **When** содержит ровно одно действие — нет "And" в When
- **Then** описывает наблюдаемый результат с точки зрения вызывающей стороны — не внутреннюю реализацию
- Формулировки user-centric: что возвращается/наблюдается, не как система это делает
- Результаты измеримы: конкретные коды ошибок, числовые пороги — не "быстро" / "корректно"
- Сценарии независимы — каждый самодостаточен, не зависит от порядка
- Один сценарий = одна проверка
- Нет тривиальных сценариев

## 4. Классификация проблем

Каждую найденную проблему классифицировать:
- **Пробел в acceptance** — поведение описано в spec, сценарий не написан
- **Неопределённость в spec** — сценарий невозможно написать, поведение не описано → нужен откат к `01-spec/03-fix`
- **Противоречие в spec** — два требования несовместимы → нужен откат к `01-spec/03-fix`
- **Проблема качества** — сценарий написан, но расплывчато или неправильно

## 5. Формирование отчёта

- Если нет проблем — "Acceptance чистый. Готово к test-write."
- Если есть — список с классификацией, объяснением, ссылкой на сценарий
- Статус: `done` если нет критичных пробелов, `failed` иначе

## Движение вперёд и назад

**Done когда:** нет замечаний категории "пробел в acceptance"; замечания по spec переданы оркестратору
**Следующий:** 03-tests/01-write
**Передаём:** report-NNN.md

**Возврат назад:**
- пробел в acceptance → `failed`, переход к 02-acceptance/03-fix
- неопределённость или противоречие в spec → `clarification`, возврат к 01-spec/03-fix (через оркестратора)
