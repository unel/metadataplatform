# Базовый чек-лист: 00-research-interview

## Контекст

- [ ] `TASK.md` прочитан
- [ ] `PROJECT.md` прочитан
- [ ] `PLAN.md` прочитан

## Подготовка

- [ ] По каждой категории отмечено что уже известно, что неясно
- [ ] Вопросы конкретные — не "расскажи про фичу", а про конкретное поведение
- [ ] Вопросы сгруппированы по темам
- [ ] Не спрашиваем то что уже есть в TASK.md

## Поведение

- [ ] Что делает фича — описание поведения получено
- [ ] Кто инициирует (пользователь / сервис / событие) — выяснено
- [ ] Входные данные и их источник — выяснены
- [ ] Ожидаемый результат и его формат — выяснен

## Мотивация

- [ ] Конкретная проблема которую решает — выяснена
- [ ] Что происходит сейчас без фичи — выяснено

## Граничные случаи

- [ ] Поведение при ошибке в зависимостях — выяснено
- [ ] Поведение при невалидных / отсутствующих входных данных — выяснено
- [ ] Поведение при частичном выполнении и сбое — выяснено

## Ограничения и приоритеты

- [ ] Что не входит в эту итерацию — зафиксировано явно
- [ ] Приоритеты при выборе — выяснены

## Качество разговора

- [ ] При неоднозначных решениях предложены 3-5 вариантов с pros/cons — не один "правильный"
- [ ] По каждому значимому требованию задан вопрос "что будет если не делать?" или "можем обойтись без?"
- [ ] По отложимым требованиям проверено "как отложить не потеряв?" — можно ли добавить позже без переработки
- [ ] Каждое подтверждённое требование зафиксировано и не оспаривалось повторно

## Техники

- [ ] Laddering применён — дошли до root cause через "почему?" и уточнили конкретику через "как?"
- [ ] Абстрактные требования конкретизированы — "быстро/надёжно/удобно" переведено в измеримое
- [ ] Конкретный прецедент запрошен — "расскажи про последний раз когда X пошло не так"
- [ ] Триангуляция применена — ключевые требования проверены с разных ракурсов
- [ ] Противоречия пойманы и подняты явно — не оставлены "на потом"

## Синтез

- [ ] Понимание сформулировано как зеркало и подтверждено пользователем
- [ ] Правки пользователя зафиксированы
- [ ] Явно выделено что искать в web-research

## Отчёт

- [ ] Ответы записаны дословно, без интерпретаций
- [ ] Отчёт actionable — Бо и Танк могут работать без дополнительных вопросов

## Движение дальше

- [ ] Понимание подтверждено пользователем
- [ ] Явно выделено что искать в web-research (для 00-research/02-web)
- [ ] Если данных недостаточно — записан `clarification`, пользователь уведомлён
