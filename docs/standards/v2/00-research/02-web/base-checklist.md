# Базовый чек-лист: 00-research-web

## Контекст

- [ ] Interview report прочитан — понятно что ищем
- [ ] `TASK.md` и `PROJECT.md` прочитаны — понятен стек и ограничения

## Поиск

- [ ] Поисковые запросы сформулированы до начала поиска (не хаотично)
- [ ] Найдены похожие реализации (≥ 2-3 разных подхода)
- [ ] Исследованы failure modes — что шло не так у других
- [ ] Исследованы ограничения используемых технологий/библиотек
- [ ] Исследованы устоявшиеся паттерны и их обоснование

## Источники

- [ ] Каждый факт в report-NNN.md сопровождается URL источника
- [ ] Источники достаточно авторитетны (официальная документация, известные блоги, реальные post-mortems — не случайные stackoverfow-ответы без обоснования)
- [ ] `searches.md` создан если запросов было много
- [ ] `sources.md` создан если источников было много

## Отчёт

- [ ] Отчёт actionable — Танк может писать spec без дополнительного поиска
- [ ] Явно выделено что влияет на дизайн фичи
- [ ] Явно выделено что нужно учесть при реализации
- [ ] Нет выводов без источника — каждое утверждение трассируемо

## Движение дальше

- [ ] Каждый факт имеет URL источника
- [ ] Выводы для spec и реализации явно сформулированы (для 01-spec/01-write)
- [ ] Если возникли вопросы к пользователю — записан `clarification`, оркестратор уведомлён
