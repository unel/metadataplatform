# 01-spec: Спецификация

## Что делает группа

Танк пишет спеку на основе research reports, Гримм ревьюит, Танк фиксит. Спека — это "что и зачем", не "как". Выходной артефакт: `spec.md` прошедший ревью.

## Когда входить

После `00-research` — или сразу, если контекст уже есть.

## Процессы

| Процесс | Исполнитель | Что делает |
|---|---|---|
| `01-write` | Танк | Пишет `spec.md` по research reports |
| `02-review` | Гримм | Ревьюит на полноту, точность, отсутствие implementation details |
| `03-fix` | Танк | Фиксит по замечаниям Гримма |

Цикл `02-review → 03-fix` повторяется до `pass`.

## Навигация

| Условие | Следующий шаг |
|---|---|
| `02-review` вернул `pass` | `02-acceptance/01-write` |
| `02-review` вернул `fail` | `01-spec/03-fix` |
