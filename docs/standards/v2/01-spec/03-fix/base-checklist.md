# Базовый чек-лист: 01-spec-fix

## Контекст

- [ ] Актуальная spec прочитана (из `03-fix/report-*.md` или `01-write/report-*.md`)
- [ ] Замечания прочитаны из последнего `02-review/report-*.md`
- [ ] Все CR-N из замечаний пронумерованы и покрыты в плане

## Согласование

- [ ] План исправлений показан пользователю до начала изменений
- [ ] Подтверждение получено (или правки учтены)
- [ ] Замечания требующие архитектурного решения — эскалированы, не угаданы молча

## Качество исправлений

- [ ] Исправлено только то что указано в замечаниях — no scope creep
- [ ] Несмежные разделы spec не затронуты
- [ ] Обновлённая spec самосогласована (исправления не создали новых противоречий)
- [ ] Если замечание не исправлялось — в отчёте написано почему

## Отчёт

- [ ] Отчёт содержит полный обновлённый текст spec
- [ ] Changelog: для каждого CR — что изменено или почему пропущено
- [ ] Статус: `done` если все critical/warning закрыты, иначе `clarification`

## Каскад статусов

- [ ] `01-spec/02-review` → `pending` (нужен повторный проход)
- [ ] Последующие процессы → `stale` (acceptance, tests, code, docs)

## Движение дальше

- [ ] Все critical/warning замечания закрыты (или обоснованно пропущены)
- [ ] Report написан — 01-spec/02-review помечен `pending`
- [ ] Каскад статусов выполнен: последующие процессы → `stale`
- [ ] Если требуется архитектурное решение — записан `clarification`, оркестратор уведомлён
