# 03-tests/04-run: Первый прогон тестов (Red)

## Что делает

TDD Red шаг: запускает тесты до написания реализации. Все тесты должны упасть — это подтверждает что они реально проверяют код. Тест прошедший без реализации — красный флаг.

Финальный Green прогон после реализации — в `04-code/04-testing`.

## Входящие артефакты

- Тест-файлы в `tests/happy/` и `tests/adversarial/`
- `03-tests/02-review/report-NNN.md` — подтверждение что review прошёл

## Исходящие артефакты

- `03-tests/04-run/report-NNN.md` — полный вывод тест-раннера, интерпретация падений

## Навигация

| Условие | Следующий шаг |
|---|---|
| Все тесты красные (норма) | 04-code/01-write |
| Тест прошёл без реализации | 03-tests/03-fix |
| Acceptance неполный или противоречивый | `clarification` → 02-acceptance/03-fix |

## Артефакты процесса

- `base-plan.md` — инструкции для исполнителя
- `base-checklist.md` — чек-лист запуска
- `report-NNN.md` — результат прогона (иммутабельный)
