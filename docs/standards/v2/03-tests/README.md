# 03-tests: Тесты (Red)

## Что делает группа

TDD Red фаза. Кроули и Азирафаль пишут тесты по acceptance criteria **до написания кода** — adversarial и happy path раздельно. Гримм ревьюит. При первом запуске все тесты должны упасть: если тест проходит без реализации — это красный флаг.

## Ключевые правила

- `tests/happy/` — Азирафаль; `tests/adversarial/` — Кроули
- FIRST: Fast, Isolated, Repeatable, Self-validating, Timely
- ≤ 150 строк на модуль (без комментариев)
- Тестировать поведение, не реализацию
- `04-run` = Red-only; Green запуск — в `04-code/04-testing`

## Процессы

| Процесс | Исполнитель | Что делает |
|---|---|---|
| `01-write` | Кроули + Азирафаль (параллельно) | Пишут тесты по acceptance criteria |
| `02-review` | Гримм | Ревьюит на FIRST, покрытие, изоляцию |
| `03-fix` | Кроули + Азирафаль | Фиксят по замечаниям Гримма |
| `04-run` | Азирафаль | Запускает тесты — ожидает что все упадут |

## Навигация

| Условие | Следующий шаг |
|---|---|
| `04-run` подтвердил Red (все упали) | `04-code/01-write` |
| Тест прошёл без реализации | Назад в `03-fix` — тест некорректен |
| `02-review` вернул `fail` | `03-tests/03-fix` |
