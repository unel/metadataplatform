# Базовый чек-лист: 03-tests-review

## Контекст

- [ ] Тест-файлы прочитаны (`tests/happy/` и `tests/adversarial/`)
- [ ] Report из test-write прочитан
- [ ] Acceptance прочитан для проверки покрытия

## Структура

- [ ] Happy path тесты — в `tests/happy/`
- [ ] Adversarial тесты — в `tests/adversarial/`
- [ ] Каждый модуль ≤ 150 строк (без комментариев)
- [ ] Именование тестов читается как документация

## FIRST

- [ ] **Fast** — юнит тесты не обращаются к сети/БД/ФС напрямую
- [ ] **Isolated** — нет общего mutable state, каждый тест самодостаточен
- [ ] **Repeatable** — нет `time.Now()` без инъекции, нет случайности без seed
- [ ] **Self-validating** — тест сам определяет pass/fail
- [ ] Нет скипнутых тестов без явного обоснования

## Фальсифицируемость

- [ ] Если тест содержит таймаут-обёртку — внешний таймаут теста должен быть МЕНЬШЕ ожидаемого внутреннего. Иначе тест завершается по обёртке, а не по поведению системы.
- [ ] Может ли этот тест провалиться если система ничего не делает? Если нет — тест нефальсифицируем. Обязательно для тестов на негативное поведение и таймауты.

## Поведение

- [ ] Тесты проверяют наблюдаемое поведение, не внутренние вызовы
- [ ] Тесты стабильны к рефакторингу внутренней реализации

## Structured logging

- [ ] Если тест проверяет лог-вывод — используется `attrs`, не подстрока в `msg` (`msg` — человекочитаемый шаблон, `attrs` — контракт)
- [ ] Если найден паттерн в одном месте — сделан grep по файлу/пакету перед закрытием замечания

## Покрытие acceptance

- [ ] Каждый happy path сценарий покрыт тестом Азирафаля
- [ ] Каждый failure mode покрыт тестом Кроули
- [ ] Граничные условия покрыты
- [ ] НФТ тестируемые — покрыты

## Классификация проблем

- [ ] Каждая проблема классифицирована (пробел / FIRST / реализация / неопределённость в acceptance)
- [ ] Проблемы требующие отката к acceptance — отмечены явно
- [ ] Не исправлял тесты самостоятельно — только описал проблемы

## Движение дальше

- [ ] Нет критичных замечаний → report `done`, следующий 04-code/01-write
- [ ] Есть замечания к тестам → report `failed`, следующий 03-tests/03-fix
- [ ] Acceptance неполный → `clarification`, оркестратор уведомлён для отката к 02-acceptance/03-fix
