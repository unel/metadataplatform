# Базовый чек-лист: 03-tests-fix

## Контекст

- [ ] Review report прочитан — список замечаний с классификацией
- [ ] Тест-файлы с замечаниями прочитаны
- [ ] Acceptance прочитан если нужен для уточнения

## Перед фиксом

- [ ] План фикса согласован с пользователем

## Фикс

- [ ] Исправлены только замечания из review — не переписывалось больше
- [ ] Структура `tests/happy/` и `tests/adversarial/` сохранена
- [ ] Каждый модуль ≤ 150 строк (без комментариев)

## Невозможные фиксы

- [ ] Замечания которые нельзя исправить без изменения acceptance — задокументированы в report с объяснением
- [ ] `clarification` записан, оркестратор уведомлён если применимо

## Прогон

- [ ] После фикса все тесты проходят (включая незатронутые)

## Отчёт

- [ ] Список исправленных замечаний с описанием что сделано
- [ ] Список неисправленных с объяснением

## Движение дальше

- [ ] Все исправимые замечания закрыты → report написан, возврат к 03-tests/02-review
- [ ] Если требуется изменение acceptance → `clarification`, откат к 02-acceptance/03-fix
