# !important

Не используй `!important` для решения проблем специфичности — исправь специфичность.

**Почему:** `!important` — признак что каскад вышел из-под контроля. Один `!important` порождает другой. Через месяц код становится непредсказуемым.

## Когда допустимо

- Утилитарные классы которые должны всегда побеждать: `.hidden { display: none !important; }`
- CSS reset — намеренное обнуление стилей браузера
- Лучше — использовать `@layer` для управления порядком каскада без `!important`

## Плохо

```css
.card .title { color: red; }
.title { color: blue !important; } /* борьба со специфичностью */
```

## Хорошо

```css
/* исправь специфичность или используй @layer */
.title { color: blue; }
```
