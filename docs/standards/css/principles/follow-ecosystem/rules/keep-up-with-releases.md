# Как следить за обновлениями

## Источники

- **web.dev/blog** — разборы новых CSS возможностей от команды Chrome
- **developer.mozilla.org** — MDN, обновляется при появлении новых фич
- **caniuse.com** — поддержка браузерами
- **web.dev/baseline** — статус Baseline для понимания готовности фич к использованию
- **css-tricks.com** — практические примеры новых возможностей
- **Lea Verou, Una Kravets, Adam Argyle** — ключевые авторы в CSS-сообществе

## Что искать

- Новые возможности которые заменяют JS-решения (scroll-driven animations, popover API)
- Фичи перешедшие в Baseline Widely Available — можно использовать без оговорок
- Новые селекторы и псевдоклассы (:has(), :is(), :where())
- Улучшения типографики и цветов

## Паттерн для устаревших паттернов

```css
/* OUTDATED(2022): JS-решение для sticky header с тенью */
/* актуальный подход: @scroll-timeline или animation-timeline (Baseline 2023) */
/* рефакторинг: tasks/feature/scroll-driven-header */
```

## При аудите CSS

1. Ищи места где используется JS для того что теперь делает CSS
2. Проверяй не появились ли нативные альтернативы для используемых CSS-in-JS решений
3. Ищи OUTDATED комментарии
