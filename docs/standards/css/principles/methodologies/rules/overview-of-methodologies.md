# Обзор методологий

## BEM (Block Element Modifier)

Именование: `block__element--modifier`.

**Плюсы:** явная структура, читаемость, предсказуемость.
**Минусы:** длинные имена классов, многословность.
**Когда:** хорошо для библиотек компонентов и больших команд.

```css
.card { }
.card__title { }
.card__title--highlighted { }
```

## Atomic CSS / Utility-first (Tailwind-style)

Каждый класс делает одно: `flex`, `mt-4`, `text-blue-500`.

**Плюсы:** скорость разработки, минимум написанного CSS.
**Минусы:** HTML обрастает классами, трудно читать разметку.
**Когда:** хорошо для UI-компонентов с частыми вариациями.

## CUBE CSS (Composition Utility Block Exception)

Слои: глобальная композиция → утилиты → блоки → исключения.

**Плюсы:** балансирует между utility и компонентным подходом, современный.
**Минусы:** требует дисциплины в разграничении слоёв.
**Когда:** хорошо вместе с `@layer`.

## TAC (Type Atom Component)

Type — базовые HTML-элементы. Atom — мелкие переиспользуемые единицы. Component — составные блоки.

**Плюсы:** чёткая иерархия, избегает specificity wars.
**Минусы:** малоизвестна, меньше инструментов.
**Когда:** хорошо для design system с чёткой иерархией.

## SMACSS

Слои: Base → Layout → Module → State → Theme.

**Плюсы:** явное разделение ответственности, масштабируется.
**Минусы:** накладные расходы на организацию для небольших проектов.
**Когда:** хорошо для крупных проектов с несколькими темами.

## ITCSS (Inverted Triangle)

Слои от общего к специфичному: Settings → Tools → Generic → Elements → Objects → Components → Utilities.

**Плюсы:** контролируемая специфичность, хорошо работает с `@layer`.
**Минусы:** много слоёв, порог входа.
**Когда:** хорошо как основа для design system.

## CSS Modules

Локальные классы по умолчанию — компилятор генерирует уникальные имена.

**Плюсы:** полная изоляция, нет конфликтов, естественно для SvelteKit (`<style>`).
**Минусы:** сложнее переопределять снаружи.
**Когда:** компонентный подход, SvelteKit scoped styles.

---

## Рекомендация для этого проекта

Нет жёсткого выбора одной методологии. Ориентир:

- **Scoped styles** в SvelteKit — основа изоляции компонентов
- **Custom properties** — токены дизайна
- **@layer** — управление каскадом для глобальных стилей
- **Utility-классы** — минимальный набор для частых паттернов
- **BEM или CUBE** — по ситуации для сложных компонентов
