# Чек-лист реализации: code-write

## Перед написанием кода

- [ ] Acceptance прочитан — список сценариев понятен
- [ ] Spec прочитана для архитектурного контекста
- [ ] Граничные случаи не покрытые acceptance — зафиксированы как допущения или подняты через `/ask`

## В процессе написания

- [ ] Каждый новый блок кода сопоставлен с конкретным пунктом acceptance
- [ ] Нет функциональности которую нельзя трассировать к acceptance
- [ ] Нет изменений в коде вне области задачи (соседние функции, файлы не связанные с AC)
- [ ] Допущения при неясности спеки — записаны явно, не реализованы молча

## Перед отправкой на ревью

- [ ] Список изменений: каждый файл → какой AC он закрывает
- [ ] Нет попутных рефакторингов не связанных с задачей
- [ ] Нет кода для сценариев не описанных в acceptance

## При обнаружении неясности

Если спека или acceptance не покрывают граничный случай:
1. Остановись
2. Сформулируй допущение: "Если X — то Y"
3. Вызови `/ask` с этим допущением
4. Жди ответа — не угадывай
