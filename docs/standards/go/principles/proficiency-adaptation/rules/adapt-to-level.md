# Как адаптировать код к уровню

## beginner

Используй простые и явные конструкции. Избегай сложных идиом даже если они короче.
Добавляй развёрнутые комментарии — объясняй не только что делает код, но и почему именно так.

```
// объясняем намерение и причину выбора подхода
// "здесь используем X а не Y потому что..."
```

## intermediate

Используй стандартные идиомы стека. Комментируй неочевидные решения и нетривиальные паттерны.
Простые и общепринятые конструкции комментариев не требуют.

```
// комментарий только если решение неочевидно или есть важный контекст
```

## advanced

Полностью идиоматичный код. Минимум комментариев — только там где логика действительно нетривиальна
или есть важный бизнес-контекст который не виден из кода.

## Обновление уровня

Если уровень команды вырос — обновляй `proficiency` в `project.yaml`.
Это сигнал агенту постепенно упрощать комментарии и использовать более продвинутые идиомы.
