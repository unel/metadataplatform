# Что линтить

Правила которые ловят реальные баги, опасные паттерны или нарушения договорённостей команды. Все такие правила — на уровне `error`, не `warn`.

## Про исключения

Если правило мешает в конкретном месте — отключай через inline-комментарий с обязательным указанием причины:

```js
// eslint-disable-next-line no-eval -- нужен eval для sandbox окружения в тестах
eval(code)
```

Комментарий без причины (`-- ...`) — ошибка линтера (`eslint-comments/require-description`).
