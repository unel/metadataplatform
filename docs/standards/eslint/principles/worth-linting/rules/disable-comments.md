# Inline-отключения без причины

**Категория:** контроль за использованием `eslint-disable` комментариев.

**Почему:** `eslint-disable` без причины — способ замолчать линтер не думая. Обязательная причина заставляет осознанно принимать решение об исключении.

## Правило

Любой `eslint-disable` комментарий должен содержать причину через `--`:

```js
// хорошо
// eslint-disable-next-line no-eval -- sandbox для тестов, eval необходим

// плохо — линтер должен это ловить
// eslint-disable-next-line no-eval
```

## Пример правила

```json
"eslint-comments/require-description": "error"
```
