# Сложность функций

**Категория:** метрики сложности — цикломатическая сложность, глубина вложенности, число параметров, длина функции.

**Почему:** высокая сложность коррелирует с количеством багов и трудностью тестирования. Линтер даёт объективный сигнал без вкусовщины.

## Примеры правил (не исчерпывающий список)

```json
"complexity": ["error", 10],
"max-depth": ["error", 4],
"max-params": ["error", 4],
"max-lines-per-function": ["error", 50]
```

Конкретные пороги — на усмотрение команды. Главное что категория линтится.

## Про исключения

Сложный алгоритм где порог оправдан — отключи с причиной:

```js
// eslint-disable-next-line complexity -- алгоритм парсинга DSL, сложность неизбежна
function parseDSL(tokens) { ... }
```
