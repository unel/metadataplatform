# Versioning

Версионирование API с первого дня — смена версии не должна ломать существующих клиентов.

## Подходы

**URL prefix** — `/api/v1/entities` — наиболее явный, легко кэшируется.

**Header** — `Accept: application/vnd.api+json;version=1` — чище URL, сложнее в использовании.

Выбери один подход и следуй ему консистентно.

## Правила совместимости

- Добавление нового поля в ответ — не breaking change
- Удаление поля — breaking change, требует новой версии
- Изменение типа поля — breaking change
- Добавление нового эндпоинта — не breaking change
- Изменение семантики существующего эндпоинта — breaking change
