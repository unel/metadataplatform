# Валидация входных данных

Валидируй всё что приходит снаружи — HTTP запросы, аргументы CLI, сообщения из очереди, файлы.

**Почему:** внешние данные — зона недоверия. SQL injection, path traversal, XXE — все эти атаки работают через невалидированный ввод.

## Принципы

- Валидируй на входе в систему — не глубоко внутри
- Отклоняй всё что не соответствует ожидаемому формату
- Параметризованные запросы — единственный способ работы с SQL
- Не используй пользовательский ввод для построения путей файловой системы без санитизации

## Плохо

```
query = "SELECT * FROM entities WHERE type = '" + userInput + "'"
```

## Хорошо

```
query = "SELECT * FROM entities WHERE type = $1"
db.query(query, userInput)
```
