# Как следить за обновлениями

## Источники

- **svelte.dev/docs** — официальная документация, всегда актуальная
- **svelte.dev/blog** — анонсы новых версий и изменений
- **github.com/sveltejs/kit/releases** — changelog SvelteKit
- **Kevin Åberg Kultalahti, Rich Harris** — ключевые авторы, полезно следить

## Что искать

- Депрекации — что устарело и чем заменить (например: stores → runes)
- Новые API которые упрощают существующий код
- Изменения в рекомендуемых паттернах роутинга и загрузки данных
- Breaking changes при обновлении версии

## Паттерн для устаревших паттернов

Когда знаешь что паттерн устарел но рефакторинг отложен — помечай:

```svelte
<!-- OUTDATED(Svelte 4): использует readable store вместо $state -->
<!-- актуальный подход: createXxx паттерн с рунами -->
<!-- рефакторинг: tasks/feature/stores-migration -->
```

## При обновлении зависимости

1. Читай migration guide, не только changelog
2. Ищи OUTDATED комментарии в коде
3. Ищи deprecation warnings в консоли браузера и сервера
