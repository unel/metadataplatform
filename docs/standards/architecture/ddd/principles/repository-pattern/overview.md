# Repository Pattern

Store — единственная точка доступа к данным. Скрывает детали хранилища (SQL, индексы, транзакции) за единым интерфейсом. Остальные компоненты работают с доменными объектами, не с деталями БД.

Признак нарушения: компонент работает с БД напрямую минуя store, или знает о SQL-специфике (типы, колонки, транзакции).

В проекте: `cmd/store` — единственный бинарь работающий с PostgreSQL напрямую. Все остальные — через Unix socket.
