# Bounded Context

Чёткие границы между компонентами системы. Каждый компонент знает только то что нужно для своей работы — не заглядывает за свою границу и не тянет чужие зависимости.

Признак нарушения: компонент напрямую вызывает или импортирует то что принадлежит другому контексту.

В проекте: `store` не знает о `spawner`, `worker` не знает о БД, `api` не знает о деталях хранения.
