# Layered Architecture

Зависимости идут только в одном направлении: от верхних слоёв к нижним. Нижний слой не знает о верхнем. Это делает нижние слои переиспользуемыми и тестируемыми независимо.

В проекте: `domain` (бизнес-логика) → `store` (данные) → `api`/`spawner` (оркестрация) → `workers`/`hooks` (выполнение).

Признак нарушения: нижний слой импортирует или вызывает верхний.
