- [ ] Термины из доменной модели используются в именах переменных, функций, типов
- [ ] Один и тот же концепт называется одинаково во всех слоях — нет `item`/`record`/`entry` вместо `entity`
- [ ] Новый термин согласован с командой до появления в коде
- [ ] Документация и задачи используют те же термины что и код
- [ ] Нет технических суррогатов вместо доменных понятий (`data`, `obj`, `info`, `record`)
