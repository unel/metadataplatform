# Одна ответственность

Функция делает одно действие. Если её можно описать через "и" — это две функции.

**Почему:** функция с несколькими ответственностями сложнее тестировать, сложнее переиспользовать и проще сломать при изменении одной из них.

## Границы применимости

Бизнес-логика и сервисный слой. В точках оркестрации (use case, handler) допустимо координировать несколько действий — но сами действия должны быть вынесены.

## Если соблюдать рьяно

Каждое микродействие — отдельная функция. Оркестрационная логика размазывается по десяткам мест. Понять поток выполнения становится труднее чем если бы всё было в одном месте.

## Если игнорировать

Функции превращаются в "god functions" — меняешь одно, ломается другое. Тесты пишутся только для всей функции целиком, покрытие отдельных путей невозможно.

## Когда можно отступить

Точки входа (main, handler, use case) по природе координируют несколько действий — это их ответственность. Главное чтобы сами действия были вынесены в отдельные функции.

## Плохо

Функция сохраняет И уведомляет:

```
function saveAndNotify(user):
    db.save(user)
    mailer.send(user.email, "Saved!")
```

## Хорошо

Каждое действие — отдельная функция:

```
function saveUser(user):
    return db.save(user)

function notifyUser(user):
    return mailer.send(user.email, "Saved!")
```
