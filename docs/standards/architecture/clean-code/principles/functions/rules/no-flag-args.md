# Без флаговых аргументов

Не передавай булевые флаги в функцию — сделай две отдельные функции.

**Почему:** булевый флаг — явный признак что функция делает два разных действия. На месте вызова смысл флага непонятен без заглядывания в сигнатуру.

## Границы применимости

Публичные функции с поведенческими флагами. Конфигурационные структуры с булевыми полями — другой случай, там флаги допустимы.

## Если соблюдать рьяно

Proliferation функций: `renderPage`, `renderPageAsAdmin`, `renderPageAsAdminWithSidebar`. Дублирование кода между похожими функциями хуже чем один флаг.

## Если игнорировать

`process(data, true, false, true)` — что означает каждый флаг? Читатель обязан смотреть сигнатуру при каждом чтении.

## Когда можно отступить

Когда флаг — это конфигурация а не ветвление поведения: `formatDate(date, includeTime: bool)` — допустимо если обе ветки делают одно и то же с небольшим отличием.

## Плохо

Флаг меняет поведение — на вызове смысл непонятен:

```
function render(page, asAdmin: bool):
    if asAdmin -> renderAdminView(page)
    else -> renderUserView(page)

render(page, true)
```

## Хорошо

Два разных действия — две функции:

```
function renderPage(page): ...
function renderPageAsAdmin(page): ...
```
