# Не больше 3 аргументов

Функция принимает не более трёх аргументов. Больше — сигнал что нужна структура.

**Почему:** чем больше аргументов, тем сложнее вызывать функцию правильно — легко перепутать порядок, легко передать не то.

## Границы применимости

Публичные функции и методы. Внутренние вспомогательные функции с очевидным контекстом допускают чуть больше гибкости.

## Если соблюдать рьяно

Создаются структуры ради одного вызова с двумя аргументами. Это добавляет шаблонный код без реальной пользы.

## Если игнорировать

Функции с 6-8 аргументами. Вызов нечитаем — `create(true, false, nil, 0, "")`. Легко перепутать порядок булевых и числовых аргументов.

## Когда можно отступить

- Математические функции: `clamp(value, min, max)` — три аргумента оправданы и читаемы
- Функции с устоявшимися сигнатурами в конкретном стеке (middleware, обработчики событий)

## Плохо

Шесть аргументов — вызов нечитаем:

```
function createUser(name, email, role, department, age, active):
    ...

createUser("Bob", "bob@example.com", "admin", "engineering", 30, true)
```

## Хорошо

Аргументы сгруппированы в структуру:

```
struct CreateUserParams:
    name, email, role, department, age, active

function createUser(params: CreateUserParams):
    ...

createUser({ name: "Bob", email: "bob@example.com", role: "admin", ... })
```
