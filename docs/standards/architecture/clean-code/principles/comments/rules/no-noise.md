# Не добавляй шум

Не комментируй очевидное. Комментарий который не добавляет информации — это шум.

**Почему:** шумовые комментарии приучают читателя их игнорировать. Когда появляется важный комментарий — его тоже пропускают.

## Границы применимости

Весь внутренний код. В публичном API минимальная документация может быть обязательной по соглашению — даже если кажется очевидной.

## Если соблюдать рьяно

Удаляются комментарии которые всё же несут смысл для конкретного читателя. Субъективность "очевидности" зависит от опыта.

## Если игнорировать

Файлы на 30% состоят из комментариев типа `// constructor`, `// returns true if active`, `// i++`. Полезный сигнал тонет в шуме.

## Когда можно отступить

Учебный или демонстрационный код где объяснение очевидного — часть цели.

## Плохо

Комментарии дублируют имена или пересказывают синтаксис:

```
// конструктор
function newUser(name, email):
    ...

// возвращает true если пользователь активен
function isActive() -> bool:
    return user.active
```

## Хорошо

Комментарии только там где они добавляют информацию:

```
function newUser(name, email):
    ...

function isActive() -> bool:
    return user.active
```
