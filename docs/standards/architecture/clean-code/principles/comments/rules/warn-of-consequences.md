# Предупреждай о последствиях

Комментарий должен предупреждать о неочевидных последствиях, ограничениях или требованиях к вызывающему коду.

**Почему:** разработчик который не знает контекста может сломать инвариант или вызвать функцию неправильно. Предупреждение экономит часы отладки.

## Границы применимости

Функции с нетривиальными предусловиями, глобальное состояние, порядок инициализации, потокобезопасность, производительность.

## Если соблюдать рьяно

Каждая функция обвешивается предупреждениями. Читатель перестаёт их воспринимать — эффект "wolf cry".

## Если игнорировать

Разработчик вызывает функцию не в том потоке, не в том порядке, с не теми данными. Баг обнаруживается в продакшене.

## Когда можно отступить

Если ограничение очевидно из типов или имён — комментарий избыточен.

## Плохо

Нет предупреждения о критичном требовании:

```
function parseConfig(path):
    // читает файл и парсит конфиг
    ...
```

## Хорошо

Предупреждение о неочевидном требовании:

```
// Должна вызываться до любых обращений к БД —
// инициализирует пул соединений. Повторный вызов — no-op.
function parseConfig(path):
    ...
```
