# Объясняй почему, а не что

Комментарий должен раскрывать намерение или причину — не пересказывать что делает код.

**Почему:** "что" видно из кода. "Почему" — нет. Комментарий-пересказ устаревает при первом же изменении кода и начинает лгать.

## Границы применимости

Весь нетривиальный код. Особенно важно для неочевидных решений, компромиссов и обходных путей.

## Если соблюдать рьяно

Каждая строка сопровождается эссе о намерении. Комментарии становятся длиннее кода.

## Если игнорировать

Комментарии типа `// increment i` рядом с `i++`. Читатель получает ноль новой информации и привыкает игнорировать комментарии вообще.

## Когда можно отступить

Публичный API: там комментарий объясняет и что и почему — для внешнего потребителя контекст недоступен.

## Плохо

Комментарий пересказывает код:

```
// получаем пользователя по id
user = getUser(id)

// если пользователь не найден — возвращаем ошибку
if user == nil -> return error("not found")
```

## Хорошо

Комментарий объясняет намерение там где оно неочевидно:

```
// Откатываем к дефолтной локали если явная не задана —
// браузер может прислать q=0 для нежелательных локалей
locale = user.locale ?? defaultLocale
```
