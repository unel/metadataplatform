# Избегай дезинформации

Имя не должно намекать на то чем объект не является.

**Почему:** дезинформирующее имя хуже чем плохое — оно активно вводит в заблуждение. Читатель строит неверную модель и потом долго ищет баг.

## Границы применимости

Особенно критично для публичных API, параметров функций и полей структур — всего что видят другие разработчики.

## Если соблюдать рьяно

Можно уйти в излишнюю точность: `mapOfAccountsKeyedByStringIDWithPointerValues` — это уже слишком. Имя должно быть точным, но не исчерпывающим.

## Если игнорировать

Разработчик видит `accountList`, ожидает список с индексом и порядком, а получает словарь без порядка. Баги обнаруживаются поздно и с трудом.

## Когда можно отступить

Устоявшиеся соглашения важнее точности: `handler`, `middleware`, `ctx` — все знают что это такое даже без уточнений.

## Плохо

Имя говорит "это список" — но это словарь:

```
accountList = map of accounts by id
```

## Хорошо

Имя отражает реальную структуру:

```
accountsByID = map of accounts by id
accounts = collection of accounts
```
