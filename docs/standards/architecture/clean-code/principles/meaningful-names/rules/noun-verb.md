# Классы — существительные, методы — глаголы

Классы и структуры именуются существительными, методы — глаголами.

**Почему:** существительное описывает сущность, глагол — действие. Смешение путает читателя — непонятно является ли `Manager` объектом или действием.

## Границы применимости

Классы, структуры, интерфейсы — существительные. Методы, функции — глаголы. Пакеты и модули — по соглашению конкретного стека.

## Если соблюдать рьяно

Принуждает придумывать искусственные существительные там где их нет: `Executor`, `Processor`, `Manager` — размытые имена которые ничего не говорят о конкретной ответственности.

## Если игнорировать

`SaveUser`, `DeleteAccount` как имена классов. `data`, `info` как имена методов. Код становится непредсказуемым — нельзя угадать является ли идентификатор объектом или действием.

## Когда можно отступить

Функциональный стиль: функции-трансформеры часто именуются существительными (`filter`, `mapper`). Builder-паттерн: методы могут быть существительными (`user.name("Bob")`).

## Плохо

Класс назван как действие, метод — как существительное:

```
class DataManager
    function manager() -> result
```

## Хорошо

Класс — существительное, методы — глаголы:

```
class Account
    function save() -> error
    function delete() -> error
    function validate() -> bool
```
