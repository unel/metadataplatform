# Один концепт — одно слово

Для одного типа действия используй один глагол во всём проекте.

**Почему:** `fetch`, `retrieve`, `get` означают одно и то же — читатель тратит время выясняя разницу там где её нет. Консистентность снижает когнитивную нагрузку.

## Границы применимости

Особенно важно в пределах одного модуля или пакета. Между разными сервисами допустима некоторая несогласованность если они независимы.

## Если соблюдать рьяно

Попытка унифицировать всё приводит к неудобным именам: называть `authenticate` просто `get` чтобы не нарушать консистентность — хуже чем исключение.

## Если игнорировать

Читатель начинает искать разницу между `fetchUser` и `getUser` там где её нет. Новые разработчики не знают какое слово использовать и добавляют новые синонимы.

## Когда можно отступить

Когда разные слова несут реальный смысловой оттенок: `getUser` (из кэша) vs `fetchUser` (из сети) — если это различие важно и документировано.

## Плохо

Одно и то же действие — разные слова в разных местах:

```
function fetchUser(id)
function retrieveAccount(id)
function getOrder(id)
```

## Хорошо

Один глагол для одного типа действия во всём проекте:

```
function getUser(id)
function getAccount(id)
function getOrder(id)
```
