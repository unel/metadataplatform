# Форматирование — соглашение команды

Форматирование — не личный стиль. Если в проекте есть автоформаттер — используй его. Если есть соглашение — следуй ему.

**Почему:** несогласованное форматирование создаёт шум в diff'ах, затрудняет чтение и сигнализирует об отсутствии дисциплины. Мозг тратит ресурсы на разбор стиля вместо логики.

## Границы применимости

Весь код в проекте. Особенно критично в местах с частыми конфликтами: скобки, отступы, длина строки, порядок импортов.

## Если соблюдать рьяно

Бессмысленные споры о стиле заменяют обсуждение архитектуры. Любое отклонение от автоформаттера становится поводом для ревью-комментария.

## Если игнорировать

Каждый разработчик форматирует по-своему. Diff'ы содержат форматирование вместо логики. git blame бесполезен после "форматирование" коммита.

## Когда можно отступить

Автоформаттер иногда ухудшает читаемость в специфичных случаях (длинные цепочки, таблицы констант) — локальное отключение с комментарием почему.

## Плохо

Каждый файл в своём стиле, ручное форматирование вместо автоформаттера:

```
// файл A
function foo(){return bar}

// файл B
function foo()
{
    return bar;
}
```

## Хорошо

Автоформаттер настроен и запускается автоматически (pre-commit hook, CI). Весь код выглядит одинаково независимо от автора.
