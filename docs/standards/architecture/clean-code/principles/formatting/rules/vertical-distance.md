# Вертикальное расстояние

Связанные вещи располагаются рядом. Несвязанные — далеко. Пустые строки разделяют концептуальные блоки.

**Почему:** читатель воспринимает близость как связанность. Если вызов функции далеко от её определения — читатель вынужден скроллить и держать контекст в голове.

## Границы применимости

Порядок функций в файле, порядок блоков внутри функции, группировка связанных переменных.

## Если соблюдать рьяно

Попытка разместить всё "рядом" приводит к перемешиванию разных концепций в одном месте. Принцип работает на уровне концептуальной близости, а не буквальной.

## Если игнорировать

Функция вызывается в начале файла, определена в конце, зависимости разбросаны по середине. Читатель прыгает по файлу чтобы понять одно действие.

## Когда можно отступить

Публичный API: порядок может диктоваться документацией или алфавитным соглашением, а не близостью использования.

## Плохо

Связанные функции разделены несвязанными:

```
function parseRequest(r): ...
function logMetrics(m): ...      // несвязанная функция посередине
function validateRequest(r): ...  // связана с parseRequest — но далеко
```

## Хорошо

Связанные функции расположены рядом:

```
function parseRequest(r): ...
function validateRequest(r): ...

function logMetrics(m): ...
```
