# Структура файла

Публичные объявления — раньше приватных. Высокоуровневая логика — раньше деталей реализации. Читатель видит интерфейс прежде чем погружаться в детали.

**Почему:** большинство читателей хотят понять что делает модуль — не как именно он это делает. Структура файла должна отвечать на "что" раньше чем на "как".

## Границы применимости

Модули, пакеты, файлы с несколькими функциями. Небольшие файлы с одной функцией — порядок не критичен.

## Если соблюдать рьяно

Искусственное разделение публичного и приватного когда они тесно связаны ухудшает локальность — нарушает правило вертикального расстояния.

## Если игнорировать

Публичный интерфейс разбросан по файлу среди приватных деталей. Читатель не может быстро понять что предоставляет модуль.

## Когда можно отступить

Если приватная функция используется только одним публичным методом — лучше разместить её рядом (вертикальное расстояние важнее).

## Плохо

Приватные детали перемешаны с публичным интерфейсом:

```
function _buildQuery(filter): ...    // приватная деталь
function getUsers(filter): ...       // публичный метод
function _parseRow(row): ...         // приватная деталь
function createUser(data): ...       // публичный метод
```

## Хорошо

Публичные методы — сверху, детали — снизу:

```
function getUsers(filter): ...
function createUser(data): ...

function _buildQuery(filter): ...
function _parseRow(row): ...
```
