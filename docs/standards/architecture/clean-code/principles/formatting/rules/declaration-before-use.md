# Объявление близко к использованию

Переменная объявляется максимально близко к месту первого использования — не в начале функции.

**Почему:** читатель видит переменную и сразу понимает зачем она нужна. Объявление в начале функции заставляет держать её в голове пока она не понадобится.

## Границы применимости

Локальные переменные внутри функций. Менее применимо к языкам где объявление в начале блока — единственная возможность (C89).

## Если соблюдать рьяно

В языках без вывода типов появляется много мелких объявлений разбросанных по коду — читаемость ухудшается.

## Если игнорировать

Все переменные объявлены в начале функции. Читатель видит `result`, `tmp`, `err`, `buf` — и не знает зачем они нужны пока не дочитает до середины.

## Когда можно отступить

Переменные используемые в нескольких ветках условия — лучше объявить до условия. Переменные с дорогой инициализацией — объявляются один раз наверху.

## Плохо

Все переменные объявлены в начале — далеко от использования:

```
function process(data):
    result = nil
    tmp = nil
    err = nil
    count = 0

    // 20 строк логики
    ...
    result = transform(data)
    ...
```

## Хорошо

Переменная объявляется прямо перед использованием:

```
function process(data):
    // 20 строк логики без result
    ...
    result = transform(data)
    return result
```
