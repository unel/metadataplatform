# Оборачивай ошибку контекстом

При проброске ошибки вверх добавляй контекст — что именно делала функция когда случилась ошибка.

**Почему:** ошибка без контекста бесполезна при отладке. `connection refused` — где? зачем? при каком запросе? Контекст превращает загадку в понятное сообщение.

## Границы применимости

Все функции которые пробрасывают ошибку из нижнего слоя в верхний. Особенно важно на границах слоёв (репозиторий → сервис → handler).

## Если соблюдать рьяно

Цепочка контекста становится избыточной: `get user: query users: scan row: EOF`. Каждый слой добавляет контекст — итоговое сообщение перегружено.

## Если игнорировать

В логах: `sql: no rows`. Непонятно какой запрос, какой пользователь, какой endpoint. Debugging занимает часы вместо минут.

## Когда можно отступить

На верхнем уровне (handler, main) — там ошибка уже логируется или возвращается пользователю с достаточным контекстом. Дополнительная обёртка избыточна.

## Плохо

Ошибка пробрасывается без описания что происходило:

```
function getUser(id):
    user, err = db.query("SELECT * FROM users WHERE id = ?", id)
    if err != nil -> return nil, err
```

## Хорошо

Ошибка оборачивается с описанием операции:

```
function getUser(id):
    user, err = db.query("SELECT * FROM users WHERE id = ?", id)
    if err != nil -> return nil, error("get user {id}", err)
```
