# Чек-лист: тесты с логами

Использовать при написании тестов и при ревью тестов которые взаимодействуют с логами.

## При написании теста

- [ ] Тест проверяет attrs (структурированные поля), не подстроку в msg
- [ ] Хелпер для проверки лога принимает ключ и значение, не строку поиска
- [ ] Тестовый логгер предоставляет доступ к attrs отдельно от msg
- [ ] Если хелпер принимает строку — проверено что именно он ищет (msg или attrs)
- [ ] `waitForMsg(string)` или аналог — заменён на `waitForAttr(key, value)` или обоснован явно

## При ревью теста

- [ ] Нет вызовов вида `waitForMsg("field_name")` где `field_name` — это имя атрибута
- [ ] Нет `strings.Contains(logOutput, "attr_name")` без явного разбора структуры
- [ ] Если тест проверяет лог — понятно какой именно слой проверяется (msg или attrs)
- [ ] Если паттерн ошибки найден в одном месте — выполнен grep по пакету/файлу на аналогичный паттерн

## Красные флаги

- Хелпер `waitForMsg` / `containsMsg` / `hasLogLine` принимает строку без явного указания слоя
- Тест "проходит по таймауту" — признак что условие никогда не выполняется, а тест просто ждёт
- Тест проверяет лог, но тестовый логгер не имеет метода для чтения attrs
