# Test Pyramid

Тесты делятся на три уровня: юниты (много, быстрые), интеграционные (меньше, медленнее), E2E (мало, медленные). Соотношение важно — перевёрнутая пирамида делает CI медленным и ненадёжным.

В проекте: Go юниты + testcontainers для интеграционных, Playwright для E2E, Storybook для UI-компонентов.
