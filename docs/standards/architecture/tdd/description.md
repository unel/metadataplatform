# Test-Driven Development — принципы

## Test Pyramid
- **Юнит тесты** — много, быстрые, тестируют изолированную логику
- **Интеграционные** — меньше, тестируют взаимодействие компонентов (store + реальная БД)
- **E2E** — мало, тестируют критические пользовательские сценарии

В проекте: Go юниты + testcontainers для интеграционных, Playwright (chromium) для E2E, Storybook для компонентов.

## Что тестировать
- Поведение, не реализацию — тест не должен знать как работает код внутри
- Публичные интерфейсы, не приватные методы
- Граничные условия и corner-cases из acceptance.md

## Что не тестировать
- Тривиальный код (геттеры, константы)
- Детали реализации которые могут измениться
- Внешние библиотеки и фреймворки
