# Без спекулятивной обобщённости

Не делай код более общим чем требует текущая задача. Обобщение должно быть продиктовано реальной потребностью, а не предположением о будущем.

**Почему:** "может понадобиться" почти никогда не совпадает с тем что понадобится на самом деле. Спекулятивный код платит цену сложности прямо сейчас — за пользу которой может не быть никогда.

## Границы применимости

Новая функциональность, параметры функций, точки расширения, плагин-системы. Менее применимо к доменным моделям где гибкость заложена в требованиях.

## Если соблюдать рьяно

Отсутствие любой абстракции приводит к жёсткому коду который трудно расширять даже когда потребность появилась. Баланс между YAGNI и разумной гибкостью требует суждения.

## Если игнорировать

Система обрастает неиспользуемыми параметрами, интерфейсами с одной реализацией, plugin-системами без плагинов. Код сложнее читать и менять.

## Когда можно отступить

Когда расширяемость — явное требование из спеки. Когда цена добавления позже значительно выше цены добавления сейчас (например: схема БД).

## Плохо

Обобщение без текущей необходимости:

```
// сейчас нужен только один формат — но "вдруг понадобится другой"
function export(data, formatter = defaultFormatter, transformer = identity, validator = noopValidator):
    ...
```

## Хорошо

Ровно то что нужно сейчас:

```
function exportAsCSV(data):
    ...
```
