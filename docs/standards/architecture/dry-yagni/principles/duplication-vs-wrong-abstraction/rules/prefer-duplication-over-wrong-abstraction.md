# Дублирование лучше неправильной абстракции

Если абстракция требует флагов, специальных случаев и параметров чтобы покрыть все варианты — она неправильная. Лучше разойтись в дублирование и найти правильное обобщение позже.

**Почему:** неправильная абстракция растёт вместе с системой и становится всё сложнее. Каждый новый случай добавляет ещё один флаг. В итоге — god-функция которую страшно трогать.

## Границы применимости

Любой код где общая функция начинает обрастать параметрами `mode`, `type`, `isSpecialCase`.

## Если соблюдать рьяно

Любое несовпадение случаев воспринимается как сигнал к разделению. Код дублируется там где хорошая абстракция была бы уместна.

## Если игнорировать

Функция `render(data, isAdmin, isPreview, isPDF, includeFooter)` которая делает принципиально разные вещи в зависимости от комбинации флагов.

## Когда можно отступить

Когда случаи действительно схожи и отличаются только конфигурацией — параметры уместны. Граница между "конфигурация" и "разные концепции" требует суждения.

## Плохо

Абстракция обросла флагами — каждый новый случай добавляет параметр:

```
function exportData(data, format, includeHeader, skipEmpty, adminMode, legacyCompat):
    if adminMode -> ...
    if legacyCompat -> ...
    if format == "csv" -> ...
    if format == "json" -> ...
```

## Хорошо

Разные концепции — разные функции, даже если код похож:

```
function exportAsCSV(data, options): ...
function exportAsJSON(data, options): ...
function exportAdminReport(data): ...
```
