# Code Complete

Принципы из Code Complete (Steve McConnell) применимые к проекту.

## Defensive Programming
- Валидируй входные данные на границах системы (API, сокет)
- Внутри системы доверяй — не валидируй повторно то что уже проверено
- Явные preconditions — если функция требует определённого состояния, проверь его

## Complexity
- Цикломатическая сложность функции — не более 10
- Вложенность — не более 3-4 уровней
- Длина функции — помещается на экране без скролла
- Длина файла — признак проблемы если превышает ~400 строк

## Coupling & Cohesion
- **Низкое зацепление (coupling)** — компоненты минимально зависят друг от друга
  Признак нарушения: изменение одного компонента вызывает каскад изменений в других
- **Высокая связность (cohesion)** — всё что внутри модуля — относится к одной теме
  Признак нарушения: в модуле есть функции которые слабо связаны с его основной задачей
