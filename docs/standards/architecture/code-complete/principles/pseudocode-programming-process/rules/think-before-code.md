# Думай до кода

**Почему:** когда начинаешь с кода — думаешь на уровне синтаксиса и деталей реализации. Псевдокод поднимает мышление на уровень задачи. Ошибку в псевдокоде исправить за секунду; ошибку в уже написанном коде — за час рефакторинга.

## Границы применимости

Нетривиальные функции и модули. Геттер или простой маппинг — не нуждается в псевдокоде. Алгоритм обработки джобов, парсер фильтров, логика спавна — нуждается.

## Если соблюдать рьяно

Псевдокод не должен превращаться в параллельную документацию которую нужно поддерживать. Написал → перевёл в код → убрал (или оставил как комментарий если объясняет неочевидное).

## Если игнорировать

Код пишется итеративными попытками — написал, не то, переписал, снова не то. Это не TDD (там тест направляет), это просто блуждание.

## Процесс

**Шаг 1** — описать задачу словами на высоком уровне:
```
// Достать из очереди ожидающие джобы.
// Для каждого джоба найти подходящий воркер.
// Если воркер доступен — запустить, обновить статус.
// Если нет — оставить в pending.
```

**Шаг 2** — рефинировать каждый шаг:
```
// Достать из очереди ожидающие джобы:
//   - SELECT FOR UPDATE SKIP LOCKED, limit = свободные слоты
//   - Учесть лимиты по типу и подтипу из конфига

// Для каждого джоба найти подходящий воркер:
//   - Матчинг по workers/<type>.<subtype>
//   - Если файл не существует — job → failed с ошибкой

// Если воркер доступен:
//   - Обновить job.status = running
//   - Запустить процесс, передать args из payload
//   - Читать stdout построчно → отправлять в store
```

**Шаг 3** — на этом уровне детализации псевдокод переводится в код почти дословно. Edge cases (файл воркера не найден, лимиты, ошибка запуска) уже выявлены.

## Псевдокод как комментарий

Если функция сложная — псевдокод становится структурой комментариев:

```
function runPendingJobs():
  // достаём джобы с учётом лимитов
  jobs = fetchPendingWithLimits(config.limits)

  for job in jobs:
    // матчим воркер по типу
    workerPath = resolveWorker(job.worker)
    if workerPath == "":
      markFailed(job, "worker not found: " + job.worker)
      continue

    // запускаем и читаем stdout
    markRunning(job)
    runAndStream(job, workerPath)
```
