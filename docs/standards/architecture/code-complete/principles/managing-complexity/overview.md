# Managing Complexity

По McConnell, управление сложностью — **главная цель** программной конструкции. Не производительность, не элегантность, не краткость — а сложность. Всё остальное вторично.

Два типа сложности:
- **Essential complexity** — присуща самой задаче, её нельзя устранить
- **Accidental complexity** — привносится решением: плохие абстракции, лишние зависимости, запутанный поток управления. Её можно и нужно устранять.

Признак нарушения: разработчик не может удержать модуль в голове целиком — вынужден постоянно прыгать по файлам чтобы понять что происходит.
