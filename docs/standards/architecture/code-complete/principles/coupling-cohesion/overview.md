# Coupling & Cohesion

McConnell даёт подробную таксономию — не просто «coupling плохо, cohesion хорошо», а конкретные уровни от лучшего к худшему.

**Cohesion** (связность внутри модуля) — от лучшего к худшему:
1. **Functional** — модуль делает одну хорошо определённую вещь
2. **Sequential** — шаги которые должны выполняться в порядке и выход одного — вход другого
3. **Communicational** — операции над одними данными
4. **Temporal** — операции выполняются в одно время (инициализация, shutdown)
5. **Procedural** — операции выполняются в порядке, но данные не связаны
6. **Logical** — похожие операции выбираются флагом или параметром
7. **Coincidental** — операции случайно оказались в одном модуле

**Coupling** (зацепление между модулями) — от лучшего к худшему:
1. **Simple-data-parameter** — передаются только примитивные данные
2. **Simple-object** — один модуль создаёт объект другого
3. **Object-parameter** — передаётся объект вместо данных
4. **Semantic** — зависимость от внутренней семантики другого модуля
