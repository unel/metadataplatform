# overlaps

A и B частично решают одну задачу. Риск дублирования логики.

**Когда использовать:** две фичи имеют пересекающуюся функциональность —
стоит обсудить: объединить, разделить ответственность или оставить как есть.
**Обратная связь:** симметричная — если A overlaps B, то B overlaps A
**Пример:** `store/query overlaps api/commands` — оба проксируют команды в store

**Не путай с:**
- `similar-to` — фичи похожи по подходу, но не дублируют функциональность
- `partial` — часть функциональности уже реализована в другой фиче
