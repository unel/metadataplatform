# Поиск conflicts по коду (исходящие)

## Go
- два пакета открывают один и тот же сокет на запись
- несовместимые типы для одних и тех же данных (разные структуры для одного JSON)
- конкурирующие advisory locks на одни и те же ресурсы

## SvelteKit

### lib/stores/
- два стора пишут в одно и то же место с разной логикой
- несовместимые типы для одних и тех же данных

### lib/api/
- два модуля делают взаимоисключающие запросы к одному endpoint

## CSS
- конфликтующие значения одной и той же CSS custom property в разных местах
- несовместимые стили для одного и того же элемента
