# blocks

A должна быть реализована раньше B — организационная зависимость по плану.

**Когда использовать:** B нельзя начать или завершить пока не готова A,
но не обязательно по функциональным причинам во время выполнения.
**Обратная связь:** `blocked-by`
**Пример:** `store/connection blocks store/query` — нельзя писать query без соединения с БД

**Не путай с:**
- `depends-on` — функциональная зависимость во время выполнения, а не только в плане
