# extends

A строится поверх B — расширяет или специализирует её поведение.

**Когда использовать:** A не просто использует B, а добавляет поведение поверх,
переопределяет часть логики, или является частным случаем B.
**Обратная связь:** `extended-by`
**Пример:** `store/query-fts extends store/query` — FTS поиск это специализация базового query.

**Не путай с:**
- `depends-on` — A просто использует B, не расширяет её
- `similar-to` — A похожа на B, но не строится поверх неё
