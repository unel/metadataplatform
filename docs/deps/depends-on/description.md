# depends-on

A функционально зависит от B — не работает или не имеет смысла без неё.

**Когда использовать:** одна фича вызывает, читает или пишет через другую напрямую.
**Обратная связь:** `used-by` — проставляется в deps.md целевой фичи автоматически.
**Пример:** `api/entities depends-on store/query` — api читает данные через store.

**Не путай с:**
- `blocks` — организационная зависимость по плану, не функциональная
- `extends` — A строится поверх B, а не просто использует её
