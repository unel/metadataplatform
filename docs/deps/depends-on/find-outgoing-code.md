# Поиск исходящих depends-on по коду

## Go

- импорты пакетов из `internal/` других компонентов
- dial/connect к Unix сокетам (`/run/platform/*.sock`)
- использование типов и интерфейсов определённых в другом пакете
- моки или stubs других компонентов в тестах — признак зависимости

**Конфиги:**
- ссылки на пути сокетов в YAML конфигах
- переменные окружения указывающие на другой сервис

## SvelteKit

### routes/
- вызовы `fetch('/api/...')` в `+page.ts` / `+layout.ts` — зависимость от API endpoint
- использование `$page.params` — зависимость от структуры роута-родителя
- импорты из `$lib/` — зависимость от компонента, стора или типа

### lib/components/
- импорты дочерних компонентов (`import Foo from '$lib/components/Foo.svelte'`)
- использование props типов определённых в другом компоненте
- слоты которые ожидают конкретную структуру данных от родителя

### lib/stores/
- импорты других сторов (`import { foo } from '$lib/stores/foo'`)
- подписки на другой стор внутри стора (`$foo` или `foo.subscribe`)
- derived сторы — зависят от всех сторов в аргументах

### lib/api/
- какие endpoints вызывает (`/api/entities`, `/api/relations` и т.п.)
- какие типы ответов использует из `$lib/types/`

### lib/types/
- импорты типов из других модулей `$lib/types/`
- `extends` или `Pick`/`Omit` от другого типа

## CSS

- `var(--token)` — зависимость от CSS custom property определённой в другом файле
- `@import` другого CSS файла или design token файла
- использование utility классов определённых глобально
- `composes` (CSS Modules) от класса в другом файле
