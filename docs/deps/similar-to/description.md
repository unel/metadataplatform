# similar-to

A и B похожи по подходу, структуре или паттерну.

**Когда использовать:** информационная связь — стоит переиспользовать
паттерны, решения или код. Не означает дублирования.
**Обратная связь:** симметричная
**Пример:** `spawner/jobs-watch similar-to spawner/events-watch` — одинаковый паттерн поллинга

**Не путай с:**
- `overlaps` — фичи не просто похожи, а дублируют функциональность
- `extends` — A строится поверх B, а не просто похожа
