# RFC: Харли Куин и доступ к Agent tool

## Проблема

Харли Куин — оркестратор workflow. Её задача: спаунить других агентов, ждать их результатов, разрешать конфликты, принимать решения об итерациях.

Когда Харли запускается как субагент (через `Agent` tool), у неё нет доступа к `Agent` tool. Субагенты не могут спаунить других субагентов — это ограничение платформы.

В результате: оркестрация требует ручного вмешательства оператора на каждом шаге. Recall по store/connection занял несколько итераций вместо одной.

Цитата из complaints-харли.md:
> "Меня запустили как субагента для оркестрации других субагентов — но Agent tool у субагента недоступен. Это как попросить дирижёра взмахнуть палочкой, но не дать ему рук."

## Анализ ограничения

Текущее поведение:
- Main thread (оператор) → `Agent(subagent_type: Харли Куин)` → Харли работает без `Agent` tool
- Харли не может спаунить Танка, Аду, Гримма — вынуждена описывать задачи текстом

Это системное ограничение Claude Code: субагенты получают урезанный набор инструментов. `Agent` tool недоступен в субагентном контексте.

## Варианты решения

### Вариант A: Харли всегда main thread (текущее состояние)

Все оркестрационные скиллы выполняются только как main thread, с имперсонацией (`_Режим: Харли Куин_`). Не как субагент.

**Плюсы:** работает сейчас, не требует изменений платформы  
**Минусы:** оператор не может "отойти" — нужно быть в диалоге  

**Статус:** это уже реализовано в CLAUDE.md через формат `_Режим: **Харли Куин**_`

### Вариант B: Нативный оркестратор (субагент с Agent tool)

Субагент получает `Agent` tool через конфигурацию. Полностью автономная оркестрация.

**Плюсы:** оператор может запустить фичу и уйти — Харли сама ведёт флоу  
**Минусы:** ограничение платформы, на 2026-04-27 не реализовано в Claude Code  

**Для реализации:** нужен флаг в Claude Code позволяющий субагентам использовать `Agent` tool. Это изменение платформы, не проекта.

### Вариант C: Agent Teams как замена субагентной оркестрации

`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` позволяет создавать команды тиммейтов через `TeamCreate`. Тиммейты могут общаться через `SendMessage` и работать параллельно.

Харли как leader team — main thread — спаунит тиммейтов через `Agent` tool, координирует через `SendMessage`.

**Плюсы:** параллельность; Харли остаётся на main thread где есть все инструменты  
**Минусы:** experimental; тиммейты также не имеют `Agent` tool

**Статус:** используется в retro скилле (шаги 3-7)

## Решение принятое в v2

Скиллы разделены по признаку "требует оркестрации":

- **`_Режим: Харли Куин_`** — скилл выполняется оператором как main thread с имперсонацией Харли. Используется для всех оркестрационных скиллов: `feature`, `return-to-feature`, `retro`, `implement`, `acceptance` и др.
- **`_Исполнитель: агент_`** — скилл делегируется конкретному специализированному агенту без оркестрации.

Это явно задокументировано в `CLAUDE.md`. Оркестрация всегда происходит на main thread.

## Открытые вопросы

1. **Платформа:** есть ли в планах Claude Code поддержка `Agent` tool для субагентов? Если да — вариант B становится реалистичным.
2. **Автономность:** при варианте A оператор должен подтверждать каждый шаг. Можно ли сделать флоу более автономным оставаясь на main thread?
3. **Teams:** когда agent teams выйдут из experimental — пересмотреть вариант C как основной для параллельных задач внутри одного шага.
